    Chapter 03: 처음 만나는 C언어 — Hello, World!¶

    Part 1: [1권] 프로그래밍 세계에 첫 발 내딛기

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • C언어 프로그램의 기본 구조(#include, main 함수, 중괄호, 세미콜론)를 이해할 수 있다
    • printf 함수를 사용하여 원하는 문장을 화면에 출력할 수 있다
    • 코드를 작성 → 저장 → 컴파일 → 실행하는 전체 과정을 혼자 수행할 수 있다
    • 초보자가 자주 만나는 오류 메시지를 보고 원인을 찾아 고칠 수 있다

    예상 학습 시간: 약 1시간 30분 ⏱️

    📚 미리 알아야 할 것¶

    지난 장에서 우리는 이런 것들을 배웠어요:

    • 프로그래밍이란 컴퓨터에게 일을 시키기 위해 명령을 작성하는 것
    • 컴파일러는 우리가 쓴 코드를 컴퓨터가 이해할 수 있는 언어로 번역해주는 도구
    • 개발 환경(VS Code 등)을 설치하고 기본 설정을 완료한 상태

    아직 개발 환경을 설치하지 않았다면, Chapter 02로 돌아가서 설치를 먼저 완료해 주세요. 이번 장에서는 실제로 코드를 작성하고 실행해 볼 거예요!

    💡 왜 이걸 배우나요?¶

    프로그래밍을 배운다고 하면, 가장 먼저 하는 일이 있어요. 바로 "Hello, World!"를 화면에 출력하는 것입니다.

    "고작 글자 하나 출력하는 게 뭐가 대단해?" 라고 생각할 수 있어요. 하지만 이 단순한 한 줄을 화면에 띄우려면 놀랍도록 많은 일이 뒤에서 일어납니다:

    • 컴퓨터에게 "이제 명령을 줄 거야"라고 알려줘야 하고
    • "화면에 글자를 띄워줘"라는 명령을 정확한 형식으로 작성해야 하고
    • 그 명령을 컴퓨터가 이해하도록 번역(컴파일)해야 하고
    • 번역된 결과물을 실행해야 합니다

    "Hello, World!"를 성공적으로 출력했다는 것은 여러분이 위의 모든 과정을 혼자 해낸 것을 의미해요. 이것은 여러분이 프로그래머로서 첫 걸음을 진짜로 뗀 순간입니다. 🎉

    자, 그럼 시작해 볼까요?

    📚 핵심 개념¶

    개념 1: C언어 프로그램의 구조 — "편지의 형식"¶

    1) 비유로 시작 ✉️¶

    편지를 쓸 때를 떠올려 보세요. 아무리 마음이 급해도 편지에는 지켜야 할 형식이 있습니다:

    • 받는 사람 주소가 있고
    • 보내는 사람 주소가 있고
    • 편지 내용의 시작과 끝이 있고
    • 마지막에 서명을 합니다

    C언어 프로그램도 마찬가지예요. 아무리 간단한 프로그램이라도 반드시 지켜야 할 정해진 틀이 있습니다. 이 틀을 지키지 않으면 컴퓨터는 우리가 뭘 하려는 건지 전혀 이해하지 못해요.

    2) 정확한 정의¶

    자, 이제 C언어 프로그램의 가장 기본적인 형태를 보여드릴게요:

    #include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

    이 짧은 코드 안에 C언어의 핵심 구조가 모두 들어 있어요. 하나하나 분해해서 살펴봅시다.

    3) 예시로 확인 — 편지에 비유하기¶

    graph TB subgraph "C 프로그램의 구조 = 편지의 형식" A["#include &lt;stdio.h&gt;<br/>📮 준비물 챙기기<br/>(편지지와 펜을 꺼내는 것)"] B["int main(void) {<br/>📝 편지 시작<br/>(본문을 쓰기 시작)"] C["printf(...);<br/>💬 편지 본문<br/>(전하고 싶은 말)"] D["return 0;<br/>👋 마무리 인사<br/>(잘 끝났다는 신호)"] E["}<br/>📪 편지 끝<br/>(봉투를 닫는 것)"] end A --> B --> C --> D --> E

    이 그림을 머릿속에 넣어두세요. 앞으로 모든 C 프로그램은 이 구조에서 시작합니다!

    개념 2: 각 부분의 역할 — 한 줄씩 이해하기¶

    이제 위 코드를 한 줄씩 자세히 살펴볼게요. 천천히 읽어주세요. 😊

    1) #include <stdio.h> — 도구 상자 열기 🧰¶

    비유: 요리를 시작하기 전에 냉장고에서 재료를 꺼내는 것과 같아요.

    printf라는 명령(화면에 글자를 출력하는 명령)은 C언어에 기본으로 포함되어 있지 않아요. 마치 요리를 하려면 먼저 냉장고에서 재료를 꺼내야 하듯이, "나 printf를 쓸 거야, 그러니까 관련 도구를 준비해줘!" 라고 컴퓨터에게 알려주는 거예요.

    • #include: "포함시켜줘"라는 뜻
    • <stdio.h>: standard input/output, 즉 "표준 입출력" 도구 모음. 화면에 글자를 보여주거나(출력), 키보드로 입력을 받는 기능이 담겨 있어요
    • .h: header(헤더) 파일이라는 뜻. "이 파일에 유용한 도구 목록이 적혀 있으니 참고해!"라는 의미

    💡 쉽게 기억하기: stdio.h = "standard input/output의 약자". "화면에 뭔가 보여주려면 이 파일이 필요하구나!" 정도로 기억하면 돼요.

    2) int main(void) — 프로그램의 출발점 🚀¶

    비유: 영화의 "시~작!" 하고 외치는 감독의 신호와 같아요.

    모든 C 프로그램은 실행될 때 가장 먼저 main이라는 곳을 찾아갑니다. 마치 영화 촬영이 시작되면 감독이 "액션!" 하고 외치듯, 컴퓨터는 프로그램을 실행하면 "main은 어디 있지?" 하고 찾아요.

    • int: 이 함수가 끝나면 정수(integer) 숫자를 돌려준다는 뜻 (나중에 자세히 배워요, 지금은 "원래 이렇게 쓰는 거구나" 정도로 넘어가도 괜찮아요)
    • main: "주요한", "메인". 프로그램의 시작점
    • (void): "받을 것 없어"라는 뜻. 지금은 외부에서 뭔가를 받을 필요가 없으니까요. (() 만 써도 됩니다)

    3) { } — 중괄호, 울타리 🏡¶

    비유: 울타리로 둘러싸인 마당이에요. 울타리 안에 있는 것들이 이 프로그램에서 실행할 명령들이에요.

    • { (여는 중괄호): "여기서부터 명령이 시작됩니다"
    • } (닫는 중괄호): "명령이 여기서 끝납니다"

    ⚠️ 중요: 여는 중괄호 {가 있으면 반드시 닫는 중괄호 }가 있어야 해요. 마치 문을 열었으면 닫아야 하듯이요!

    4) printf("Hello, World!\n"); — 실제 명령 💬¶

    이 줄이 바로 "화면에 글자를 보여줘!"라는 명령이에요. 조금 뒤에서 아주 자세히 다룰 거예요.

    5) return 0; — 깔끔한 마무리 ✅¶

    비유: 식당에서 계산을 마치고 "잘 먹었습니다!" 하고 인사하는 것과 같아요.

    • 프로그램이 할 일을 다 했으니, "나 잘 끝냈어!" 라고 운영체제에 알려주는 것
    • 0은 "문제없이 정상 종료"를 의미해요 (0이 아닌 다른 숫자는 "뭔가 문제가 있었어"를 뜻합니다)

    6) ; — 세미콜론, 문장의 마침표 🔚¶

    비유: 한국어 문장 끝에 마침표(.)를 찍듯이, C언어에서는 명령 하나가 끝날 때마다 세미콜론(;)을 찍어요.

    C언어에서 세미콜론은 "이 명령은 여기서 끝!" 이라는 표시예요. 이걸 빠뜨리면 컴퓨터는 "명령이 아직 안 끝났나 보다..."하고 혼란에 빠집니다.

    💡 잠깐! 여기까지 이해가 잘 되셨나요? 처음 보는 것들이 많아서 조금 어지러울 수 있어요. 괜찮아요! 지금 당장 다 외울 필요 없습니다. "아, 대충 이런 구조구나" 정도면 충분해요. 직접 코드를 쳐보면서 자연스럽게 익히게 될 거예요. 😊

    개념 3: printf — 화면에 출력하기¶

    1) 비유로 시작 📢¶

    학교 방송실을 떠올려 보세요. 마이크 앞에서 "전교생 여러분, 안녕하세요!" 라고 말하면, 그 소리가 스피커를 통해 모든 교실에 울려 퍼지죠?

    printf는 바로 그 방송실 마이크와 같아요. printf에게 말을 전달하면, 그 내용이 우리의 화면(터미널)에 나타나요.

    2) 정확한 정의¶

    printf: print + formatted의 약자. "형식을 갖춰서 출력한다"는 뜻이에요.

    사용법: 

    printf("출력하고 싶은 내용");

    • printf: 함수 이름 (화면에 출력해주는 기능)
    • ( ): 괄호 안에 출력할 내용을 넣어요
    • " ": 큰따옴표 안에 보여주고 싶은 글자를 적어요
    • ;: 명령 끝! (마침표)

    3) 예시로 확인¶

    printf("Hello, World!\n");

    이 한 줄을 더 자세히 뜯어볼게요:

    graph LR A["printf"] --> B["("] B --> C["&quot;Hello, World!\\n&quot;"] C --> D[")"] D --> E[";"] style A fill:#4CAF50,color:#fff style C fill:#2196F3,color:#fff style E fill:#FF5722,color:#fff
    • 🟢 printf — "출력해줘!"라는 명령어
    • 🔵 "Hello, World!\n" — 출력할 내용 (큰따옴표로 감싸기!)
    • 🔴 ; — 명령 끝!

    그런데 여기서 \n은 뭘까요? 🤔

    개념 4: \n — 줄바꿈 문자 (이스케이프 시퀀스)¶

    1) 비유로 시작¶

    카카오톡에서 메시지를 보낼 때, Enter 키를 누르면 다음 줄로 넘어가죠? \n이 바로 그 Enter 키 역할을 해요.

    2) 정확한 정의¶

    \n: new line의 약자. "여기서 줄을 바꿔라"는 특수 명령이에요.

    \(역슬래시)로 시작하는 특수 문자를 이스케이프 시퀀스(escape sequence)라고 부르는데, 지금은 \n만 확실히 기억하면 돼요!

    💡 용어 풀이: "이스케이프(escape)"는 "탈출하다"라는 뜻이에요. 일반 글자의 세계에서 탈출해서 특별한 의미를 가지는 문자라는 뜻이죠.

    3) 예시로 확인¶

    \n이 있을 때와 없을 때를 비교해 볼까요?

    \n 없이 두 줄 출력하면: 

    printf("안녕하세요");
printf("반갑습니다");
    화면 결과: 
    안녕하세요반갑습니다
    (😱 띄어쓰기도 없이 한 줄에 다닥다닥 붙어버려요!)

    \n을 넣으면: 

    printf("안녕하세요\n");
printf("반갑습니다\n");
    화면 결과: 
    안녕하세요
반갑습니다
    (✅ 깔끔하게 줄이 바뀌었어요!)

    💡 잠깐! \n은 화면에 직접 보이지 않아요. 화면에 \n이라고 뜨는 게 아니라, 줄바꿈이라는 동작을 수행하는 거예요. 마치 Enter 키를 누르면 키보드에 있는 "Enter"라는 글자가 화면에 찍히지 않는 것과 같아요!

    🔨 따라하기 — 나의 첫 C 프로그램 작성하기¶

    드디어 직접 코드를 써볼 시간이에요! 🎉 차근차근 따라와 주세요. 어렵지 않아요.

    Step 1: 새 파일 만들기 📄¶

    목표: C언어 코드를 작성할 파일을 만듭니다.

    따라하기:

    1. VS Code를 실행하세요.
    2. 이전 장에서 만든 작업 폴더(예: c_study)를 열어주세요.
    3. 새 파일을 만드세요:
    4. 왼쪽 탐색기 패널에서 📄 새 파일 아이콘을 클릭하거나
    5. 키보드 단축키 Ctrl + N (Mac: Cmd + N)
    6. 파일 이름을 hello.c 로 저장하세요.
    7. Ctrl + S (Mac: Cmd + S)를 눌러 저장
    8. 파일 이름 입력란에 hello.c 입력

    ⚠️ 주의: 파일 확장자가 반드시 .c 여야 해요! .c가 C언어 소스 파일이라는 표시예요. hello.txt나 hello만 쓰면 안 됩니다!

    ✅ 체크포인트: 왼쪽 탐색기에 hello.c 파일이 보이나요? 파일 아이콘이 C언어 표시로 바뀌었다면 성공이에요!

    Step 2: 코드 입력하기 ⌨️¶

    목표: 첫 번째 C 프로그램을 직접 타이핑합니다.

    따라하기:

    아래 코드를 한 글자도 빠짐없이, 정확하게 타이핑해 주세요. 복사-붙여넣기보다 직접 타이핑하는 것을 강력히 추천해요! 손이 기억해야 나중에 자연스럽게 쓸 수 있거든요. 💪

    #include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

    타이핑할 때 꼭 확인해야 할 것들:

    • 1줄: #include 앞에 #(샵) 기호가 있어요. <와 >도 빠뜨리지 마세요
    • 3줄: int 다음에 한 칸 띄우고 main을 써요
    • 3줄: { (여는 중괄호)를 잊지 마세요
    • 4줄: 앞에 공백 4칸(또는 Tab 한 번)을 넣어주세요. 이걸 들여쓰기(indentation)라고 해요
    • 4줄: printf 안의 내용은 큰따옴표(") 로 감싸야 해요
    • 4줄, 5줄: 줄 끝에 세미콜론(;)이 있어요!
    • 6줄: } (닫는 중괄호)로 마무리

    💡 들여쓰기는 왜 하나요? 중괄호 { } 안에 있는 코드를 안쪽으로 밀어 쓰면, "이 코드는 main 함수에 속해있구나!"라는 것을 한눈에 알 수 있어요. 컴퓨터는 들여쓰기를 신경 쓰지 않지만, 사람이 읽기 좋게 하기 위해 꼭 해주는 것이에요.

    ✅ 체크포인트: 코드를 다 입력했으면 Ctrl + S (Mac: Cmd + S)로 저장하세요! 저장하지 않으면 컴파일할 수 없어요.

    Step 3: 컴파일하기 🔧¶

    목표: 우리가 쓴 코드를 컴퓨터가 이해할 수 있는 형태로 번역합니다.

    컴파일이란? 이전 장에서 배웠듯이, 우리가 쓴 C언어(사람의 언어)를 컴퓨터가 실행할 수 있는 기계어로 번역하는 과정이에요.

    따라하기:

    1. VS Code에서 터미널을 열어주세요.
    2. 상단 메뉴: 터미널 → 새 터미널
    3. 또는 단축키: Ctrl + ` (숫자 1 왼쪽의 키)
    4. 터미널에 아래 명령어를 입력하고 Enter를 누르세요:

    Windows (MinGW 사용 시): 

    gcc hello.c -o hello

    Mac / Linux: 

    gcc hello.c -o hello

    이 명령어의 의미:

    • gcc: C언어 컴파일러의 이름 (GNU C Compiler)
    • hello.c: 번역할 소스 파일
    • -o hello: output(출력)의 약자. 번역 결과물의 이름을 hello로 해달라는 뜻

    Step 4: 실행하기 ▶️¶

    목표: 번역된 프로그램을 실행해서 결과를 확인합니다.

    따라하기:

    터미널에 아래 명령어를 입력하고 Enter를 누르세요:

    Windows: 

    .\hello

    Mac / Linux: 

    ./hello

    화면에 이렇게 나타나야 해요: 

    Hello, World!

    ✅ 체크포인트: Hello, World!가 화면에 보이나요?

    여러분은 방금 인생 첫 C 프로그램을 만들고 실행했어요! 이 순간이 바로 프로그래머로서의 첫걸음입니다. 정말 대단해요! 👏

    💡 전 세계의 거의 모든 프로그래머가 이 "Hello, World!"로 프로그래밍을 시작했어요. 여러분도 이제 그 행렬에 합류한 거예요!

    Step 5: 나만의 문장 출력하기 ✨¶

    목표: 코드를 수정해서 원하는 문장을 출력해봅니다.

    이제 "Hello, World!" 대신 내가 원하는 문장을 출력해 볼까요?

    따라하기:

    1. hello.c 파일을 열어주세요.
    2. printf 안의 내용을 바꿔보세요:
    #include <stdio.h>

int main(void) {
    printf("안녕하세요! 저는 C언어를 배우고 있어요!\n");
    return 0;
}
    1. 저장 → 컴파일 → 실행: 
      gcc hello.c -o hello
.\hello
      (Mac/Linux는 ./hello)

    화면 결과: 

    안녕하세요! 저는 C언어를 배우고 있어요!

    이번에는 여러 줄을 출력해 볼까요?

    #include <stdio.h>

int main(void) {
    printf("===========================\n");
    printf("  나의 자기소개\n");
    printf("===========================\n");
    printf("이름: 홍길동\n");
    printf("취미: 프로그래밍 배우기\n");
    printf("목표: C언어 마스터!\n");
    return 0;
}

    화면 결과: 

    ===========================
  나의 자기소개
===========================
이름: 홍길동
취미: 프로그래밍 배우기
목표: C언어 마스터!

    🎯 미니 도전! 위 코드에서 이름, 취미, 목표를 여러분 자신의 정보로 바꿔서 실행해 보세요! 코드를 수정하고 저장 → 컴파일 → 실행하는 과정이 점점 자연스러워질 거예요.

    Step 6: 일부러 오류 만들어 보기 🐛¶

    목표: 일부러 실수를 만들어서 오류 메시지를 경험하고, 두려움을 없앱니다.

    프로그래밍에서 오류(에러)는 절대로 나쁜 것이 아니에요. 오류 메시지는 컴퓨터가 "여기에 문제가 있어요!"라고 친절하게 알려주는 힌트예요. 일부러 오류를 만들어 보면, 나중에 실수했을 때 당황하지 않을 수 있어요.
