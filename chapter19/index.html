    챕터 19: 내 코드 다듬기 — 자주 틀리는 오류 모음과 디버깅 팁¶

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • C언어 초보자가 자주 만나는 대표 오류 10가지를 보고 즉시 원인을 파악할 수 있다
    • 컴파일러가 출력하는 에러 메시지를 읽고 문제 위치를 추정할 수 있다
    • printf를 활용한 '프린트 디버깅'으로 논리 오류를 직접 찾아낼 수 있다
    • 내가 만든 프로젝트 코드를 스스로 점검하고 다듬는 습관을 들일 수 있다

    예상 학습 시간: 약 1시간 30분

    📚 미리 알아야 할 것¶

    지금까지 18개 챕터를 거치며 변수, 조건문, 반복문, 배열, 함수, 구조체까지 배웠고, 게임 프로젝트도 직접 만들어 보았습니다. 그 과정에서 "왜 안 되지?", "어디가 틀린 거지?" 하며 고민한 적이 분명 있을 거예요.

    이번 장은 그 모든 '막힘'의 순간을 정리하는 시간입니다. 새로운 문법을 배우는 것이 아니라, 지금까지 배운 것을 더 단단하게 만드는 시간이에요.

    💡 왜 이걸 배우나요?¶

    프로그래밍에서 오류는 실력 부족의 증거가 아닙니다. 전문 개발자도 매일 오류를 만나고, 매일 고칩니다. 차이가 있다면 딱 하나예요:

    🎯 "오류를 빨리 찾고, 빨리 고치는 능력"

    이 능력을 디버깅(debugging) 능력이라고 합니다. "버그(bug, 벌레)"를 잡는다는 뜻이죠. 실제로 1940년대에 컴퓨터 안에 나방이 끼어서 오작동한 것이 "버그"라는 용어의 유래입니다. 🦟

    이 장을 마치면 앞으로 혼자 코딩할 때 에러 메시지가 더 이상 무섭지 않게 될 거예요. 오히려 "아, 이거구나!" 하고 금방 해결하는 자신을 발견하게 될 겁니다.

    📚 핵심 개념¶

    이 장에서 집중할 핵심 개념은 4가지입니다:

    1. 컴파일 에러 vs 런타임 에러 vs 논리 에러 — 오류의 세 가지 종류
    2. 에러 메시지 읽는 법 — 컴파일러가 주는 단서 해독하기
    3. 대표 오류 10가지 — 증상, 원인, 해결법 총정리
    4. 프린트 디버깅 — printf로 내 코드 속을 들여다보기

    개념 1: 오류의 세 가지 종류¶

    🎭 비유로 시작¶

    요리에 비유해 볼게요.

    • 컴파일 에러: 레시피에 "달걀을 &@개 넣으세요"라고 적혀 있어서 아예 요리를 시작할 수 없는* 상황. 레시피(코드) 자체에 문법 실수가 있는 거예요.
    • 런타임 에러: 레시피대로 요리를 시작했는데, 중간에 냉장고에 달걀이 없어서 멈추는 상황. 실행 중에 예상치 못한 문제가 생긴 거예요.
    • 논리 에러: 요리가 완성되긴 했는데, 설탕 대신 소금을 넣어서 맛이 이상한 상황. 프로그램은 돌아가지만 결과가 틀린 거예요.

    📖 정확한 정의¶

    • 컴파일 에러(Compile Error): C 문법 규칙을 어겨서 컴파일러가 실행 파일을 만들 수 없는 오류. 프로그램이 아예 실행되지 않습니다.
    • 런타임 에러(Runtime Error): 프로그램 실행 중에 발생하는 오류. 갑자기 프로그램이 멈추거나 비정상 종료됩니다.
    • 논리 에러(Logic Error): 프로그램은 정상적으로 실행되지만, 결과가 의도와 다른 오류. 가장 찾기 어렵습니다.

    💡 예시로 확인¶

    // 1. 컴파일 에러 — 세미콜론 누락
int main() {
    int a = 10   // ← 세미콜론이 없어요!
    return 0;
}

// 2. 런타임 에러 — 0으로 나누기
int main() {
    int a = 10;
    int b = 0;
    int c = a / b;  // ← 실행하면 프로그램이 죽어요!
    return 0;
}

// 3. 논리 에러 — 의도와 다른 결과
int main() {
    int price = 1000;
    int discount = price * 0.1;  // 100원 할인 의도
    int result = price * discount; // ← +가 아니라 *를 썼어요!
    // result가 100,000이 됨 (의도는 900)
    return 0;
}

    💡 잠깐! 컴파일 에러는 컴파일러가 알려주니까 비교적 고치기 쉽고, 논리 에러는 아무도 알려주지 않으니까 가장 어렵습니다. 그래서 디버깅 기법이 중요한 거예요!

    개념 2: 에러 메시지 읽는 법¶

    🎭 비유로 시작¶

    에러 메시지는 의사의 진단서와 같아요. 처음엔 영어라서 무섭게 보이지만, 읽는 법만 알면 "어디가 아픈지" 정확히 알 수 있습니다.

    📖 정확한 정의¶

    컴파일러(gcc 등)가 출력하는 에러 메시지는 일정한 형식을 따릅니다:

    파일이름:줄번호:열번호: error: 설명

    💡 예시로 확인¶

    이런 코드를 컴파일했다고 해볼게요:

    #include <stdio.h>

int main() {
    int score = 85
    printf("점수: %d\n", score);
    return 0;
}

    컴파일러가 이런 메시지를 출력합니다:

    main.c:4:20: error: expected ';' before 'printf'

    이걸 분해해 볼게요:

    • main.c → 문제가 있는 파일 이름
    • 4 → 4번째 줄 근처에 문제가 있다
    • 20 → 그 줄의 20번째 글자 근처
    • error → 에러다 (warning은 경고, 실행은 됨)
    • expected ';' before 'printf' → "printf 앞에 세미콜론이 있어야 하는데 없다"

    개념 3: 대표 오류 10가지¶

    이제 C언어 입문자가 가장 자주 만나는 오류 10가지를 하나씩 살펴볼게요. 각 오류마다 증상 → 원인 → 해결법 순서로 정리합니다.

    🐛 오류 1: 세미콜론(;) 누락¶

    증상: expected ';' 에러 메시지

    원인: C언어에서 모든 문장은 세미콜론으로 끝나야 하는데, 빠뜨린 것

    // ❌ 잘못된 코드
int a = 10
int b = 20;

// ✅ 올바른 코드
int a = 10;
int b = 20;

    해결법: 에러가 가리키는 줄과 바로 윗줄의 끝에 세미콜론이 있는지 확인하세요.

    🐛 오류 2: 중괄호 { } 짝 불일치¶

    증상: expected '}' 또는 파일 끝에서 갑자기 에러 발생

    원인: 여는 중괄호 {를 열었는데 닫는 중괄호 }를 빠뜨린 것 (또는 그 반대)

    // ❌ 잘못된 코드
int main() {
    if (score > 60) {
        printf("합격!\n");
    // ← 닫는 중괄호 하나가 빠졌어요!
    return 0;
}

// ✅ 올바른 코드
int main() {
    if (score > 60) {
        printf("합격!\n");
    }  // ← 여기!
    return 0;
}

    해결법: 들여쓰기를 일관되게 하면 짝이 안 맞는 것을 눈으로 쉽게 찾을 수 있어요. {를 쓸 때 바로 }도 먼저 쓰고 그 안에 코드를 넣는 습관을 들여보세요.

    🐛 오류 3: 변수 미선언 (선언 안 하고 사용)¶

    증상: undeclared identifier 또는 '변수명' undeclared

    원인: 변수를 만들지 않고 바로 사용하거나, 변수 이름을 오타 낸 것

    // ❌ 잘못된 코드
int main() {
    socre = 85;  // score를 socre로 오타!
    printf("%d\n", score);
    return 0;
}

// ✅ 올바른 코드
int main() {
    int score = 85;  // 선언 + 올바른 이름
    printf("%d\n", score);
    return 0;
}

    해결법: 변수 이름의 철자를 꼼꼼히 확인하세요. C언어는 대소문자를 구분하므로 Score와 score는 다른 변수입니다!

    🐛 오류 4: 자료형 불일치 (printf 서식 지정자)¶

    증상: 출력 결과가 이상한 숫자로 나오거나, 경고(warning) 메시지 발생

    원인: 변수의 자료형과 printf의 서식 지정자가 맞지 않는 것

    // ❌ 잘못된 코드
float average = 85.5;
printf("평균: %d\n", average);  // float인데 %d(정수)를 사용!
// 출력: 평균: 0  (또는 이상한 숫자)

// ✅ 올바른 코드
float average = 85.5;
printf("평균: %.1f\n", average);  // float에는 %f 사용!
// 출력: 평균: 85.5

    해결법: 자료형과 서식 지정자를 짝맞춰 기억하세요.

    • int → %d
    • float → %f
    • double → %lf (scanf에서) / %f (printf에서)
    • char → %c
    • 문자열(char[]) → %s

    🐛 오류 5: 대입(=)과 비교(==) 혼동¶

    증상: 조건문이 항상 참(true)으로 실행되거나, 변수 값이 의도치 않게 바뀜

    원인: 비교해야 할 곳에서 =(대입)을 사용한 것

    // ❌ 잘못된 코드
int answer = 0;
if (answer = 1) {  // 대입! answer가 1이 되어버림
    printf("정답!\n");
}
// 항상 "정답!"이 출력됨

// ✅ 올바른 코드
int answer = 0;
if (answer == 1) {  // 비교! answer가 1인지 확인
    printf("정답!\n");
}
// answer가 0이므로 출력되지 않음

    해결법: 조건문 안에서는 반드시 ==(등호 두 개)를 사용하세요. 이 실수는 컴파일러가 에러로 잡아주지 않는 경우가 많아서 특히 주의해야 해요!

    💡 잠깐! 여기까지 5개 오류를 봤어요. 절반 왔습니다! 이런 오류들을 "아, 나도 겪었어!" 하고 떠올리셨다면, 이미 성장하고 있는 거예요. 나머지 5개도 한번 살펴볼까요? 😊

    🐛 오류 6: 무한 루프¶

    증상: 프로그램이 멈추지 않고 계속 같은 내용을 출력하거나, 아무 반응이 없음

    원인: 반복문의 종료 조건이 영원히 만족되지 않는 것

    // ❌ 잘못된 코드 — i가 영원히 증가하지 않음
int i = 0;
while (i < 10) {
    printf("%d\n", i);
    // i++; 를 깜빡했어요! i는 영원히 0
}

// ❌ 잘못된 코드 — 종료 조건 방향이 반대
for (int i = 10; i > 0; i++) {  // i가 증가하니 영원히 > 0
    printf("%d\n", i);
}

// ✅ 올바른 코드
for (int i = 0; i < 10; i++) {  // i가 증가하여 10이 되면 멈춤
    printf("%d\n", i);
}

    해결법: 반복문을 쓸 때 3가지를 꼭 확인하세요 — ①시작값, ②종료조건, ③변화식. 프로그램이 멈추지 않으면 Ctrl+C로 강제 종료할 수 있어요.

    🐛 오류 7: 배열 인덱스 범위 초과¶

    증상: 이상한 값이 출력되거나, 프로그램이 갑자기 종료(세그먼테이션 오류)

    원인: 배열의 크기를 넘어서는 인덱스에 접근한 것

    // ❌ 잘못된 코드
int scores[5] = {90, 85, 78, 92, 88};
printf("%d\n", scores[5]);  // 인덱스 0~4만 유효! 5는 범위 초과!

// ✅ 올바른 코드
int scores[5] = {90, 85, 78, 92, 88};
printf("%d\n", scores[4]);  // 마지막 원소는 인덱스 4

    해결법: 배열 크기가 N이면 사용 가능한 인덱스는 0부터 N-1까지라는 것을 항상 기억하세요!

    🐛 오류 8: scanf에서 & 빠뜨리기¶

    증상: 프로그램이 비정상 종료되거나, 입력한 값이 변수에 저장되지 않음

    원인: scanf는 변수의 주소를 필요로 하는데, 주소 연산자 &를 빠뜨린 것

    // ❌ 잘못된 코드
int age;
scanf("%d", age);  // & 빠졌어요!

// ✅ 올바른 코드
int age;
scanf("%d", &age);  // &age로 주소를 전달

// 💡 예외: 문자열(char 배열)은 배열 이름 자체가 주소이므로 & 불필요
char name[20];
scanf("%s", name);  // & 없어도 됨!

    해결법: scanf를 쓸 때 변수 앞에 &가 있는지 꼭 확인하세요. 단, 문자열 배열은 예외입니다.

    🐛 오류 9: #include 누락¶

    증상: implicit declaration of function 'printf' 또는 함수를 못 찾는 에러

    원인: 사용하려는 함수가 들어있는 헤더 파일을 포함하지 않은 것

    // ❌ 잘못된 코드
int main() {
    printf("안녕하세요!\n");  // stdio.h를 안 넣었어요!
    return 0;
}

// ✅ 올바른 코드
#include <stdio.h>  // ← 이 줄이 꼭 필요해요!

int main() {
    printf("안녕하세요!\n");
    return 0;
}

    해결법: 자주 쓰는 헤더 파일을 기억해두세요.

    • printf, scanf → #include <stdio.h>
    • strlen, strcmp → #include <string.h>
    • rand, srand → #include <stdlib.h>
    • time → #include <time.h>

    🐛 오류 10: 함수 호출 시 괄호 또는 인자 실수¶

    증상: too few arguments, too many arguments, 또는 결과값이 이상함

    원인: 함수를 호출할 때 매개변수 개수가 맞지 않거나, 순서가 틀린 것

    // ❌ 잘못된 코드
int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(10);  // 인자가 2개 필요한데 1개만 전달!
    return 0;
}

// ✅ 올바른 코드
int main() {
    int result = add(10, 20);  // 인자 2개를 올바르게 전달
    return 0;
}

    해결법: 함수를 호출하기 전에 해당 함수의 매개변수가 몇 개인지, 어떤 자료형인지 확인하세요.

    개념 4: 프린트 디버깅¶

    🎭 비유로 시작¶

    미로를 탈출한다고 상상해 보세요. 눈을 감고 가면 어디를 지나왔는지 알 수 없지만, 매 갈림길마다 표시를 남기면 어디서 잘못 꺾었는지 알 수 있어요.

    프린트 디버깅은 코드의 중간중간에 printf를 넣어서 "여기까지 왔어요", "지금 이 변수의 값은 OO이에요" 라고 표시를 남기는 것입니다.

    📖 정확한 정의¶

    프린트 디버깅(Print Debugging): 프로그램의 실행 흐름이나 변수 값을 확인하기 위해 printf문을 코드 사이사이에 임시로 삽입하는 디버깅 기법. 가장 기초적이지만 매우 효과적인 방법입니다.

    💡 예시로 확인¶

    1부터 N까지의 합을 구하는 프로그램인데, 결과가 이상하다고 해볼게요:

    #include <stdio.h>

int main() {
    int n = 5;
    int sum = 0;

    for (int i = 0; i <= n; i++) {
        sum = i;  // 🤔 뭔가 이상한데... += 이어야 하는데 =로 썼어요!
    }

    printf("1부터 %d까지의 합: %d\n", n, sum);
    // 출력: 1부터 5까지의 합: 5  (정답은 15인데!)
    return 0;
}

    결과가 5로 나왔어요. 15가 나와야 하는데 말이죠. 이때 프린트 디버깅을 적용하면:

    #include <stdio.h>

int main() {
    int n = 5;
    int sum = 0;

    for (int i = 0; i <= n; i++) {
        printf("[디버그] i=%d, 연산 전 sum=%d\n", i, sum);  // 👈 디버깅용
        sum = i;
        printf("[디버그] i=%d, 연산 후 sum=%d\n", i, sum);  // 👈 디버깅용
    }

    printf("1부터 %d까지의 합: %d\n", n, sum);
    return 0;
}

    출력 결과: 

    [디버그] i=0, 연산 전 sum=0
[디버그] i=0, 연산 후 sum=0
[디버그] i=1, 연산 전 sum=0
[디버그] i=1, 연산 후 sum=1
[디버그] i=2, 연산 전 sum=1
[디버그] i=2, 연산 후 sum=2
[디버그] i=3, 연산 전 sum=2
[디버그] i=3, 연산 후 sum=3
[디버그] i=4, 연산 전 sum=3
[디버그] i=4, 연산 후 sum=4
[디버그] i=5, 연산 전 sum=4
[디버그] i=5, 연산 후 sum=5
1부터 5까지의 합: 5

    이 출력을 보면 바로 알 수 있어요! sum이 누적되지 않고 매번 i 값으로 덮어씌워지고 있네요. sum = i가 아니라 sum += i여야 합니다!

    💡 프린트 디버깅의 핵심 규칙 3가지: 1. [디버그]처럼 접두어를 붙여서 일반 출력과 구분하세요 2. 변수명과 값을 함께 출력하세요 (어떤 변수가 어떤 값인지 명확하게) 3. 문제를 찾았으면 디버깅용 printf는 꼭 삭제하세요
