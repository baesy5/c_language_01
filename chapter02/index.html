    내 컴퓨터에 작업실 꾸미기 — 개발 환경 설치¶

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • C언어 개발에 필요한 IDE(Visual Studio Code)를 설치할 수 있다
    • 컴파일러가 무엇인지 이해하고, 정상 설치 여부를 확인할 수 있다
    • 간단한 테스트 코드를 입력하고 실행하여 결과를 확인할 수 있다
    • 설치 과정에서 자주 발생하는 오류를 스스로 해결할 수 있다

    예상 학습 시간: 약 1시간 30분

    📚 미리 알아야 할 것¶

    1장에서 우리는 프로그래밍이 "컴퓨터에게 일을 시키는 명령서를 작성하는 것"이라고 배웠어요. C언어라는 프로그래밍 언어를 선택한 이유도 함께 알아봤죠.

    이번 장에서는 드디어 실제로 코드를 작성하고 실행할 수 있는 환경을 만들어 볼 거예요. 컴퓨터에 아무것도 설치하지 않은 상태에서 시작하니, 걱정하지 마세요! 😊

    💡 왜 이걸 배우나요?¶

    요리를 배우고 싶다고 상상해 보세요. 아무리 훌륭한 레시피를 알고 있어도, 주방이 없으면 요리를 시작할 수 없겠죠? 도마, 칼, 냄비, 가스레인지... 이런 도구들이 갖춰진 주방이 있어야 비로소 레시피를 따라 요리를 만들 수 있어요.

    프로그래밍도 마찬가지예요. 우리가 C언어로 코드를 작성하려면:

    • 코드를 편하게 적을 수 있는 편집기 (도마와 칼 같은 것)
    • 코드를 컴퓨터가 이해하는 언어로 번역해 주는 컴파일러 (가스레인지 같은 것)

    이 두 가지가 반드시 필요합니다. 이번 장에서 이 "프로그래밍 주방"을 여러분의 컴퓨터에 차려볼 거예요!

    🎉 좋은 소식! 이 모든 도구는 무료입니다. 돈 한 푼 들이지 않고 전문 개발자와 같은 환경을 갖출 수 있어요!

    📚 핵심 개념¶

    이번 장에서 꼭 알아야 할 개념은 딱 4가지예요. 하나씩 차근차근 알아볼게요.

    개념 1: IDE (통합 개발 환경) 🏠¶

    비유로 시작해요

    여러분이 글을 쓴다고 상상해 보세요. 메모장에도 글을 쓸 수 있지만, 한글(HWP)이나 워드(Word)를 쓰면 훨씬 편하죠? 맞춤법 검사도 해주고, 글꼴도 바꿔주고, 페이지 설정도 알아서 해주니까요.

    프로그래밍에서 IDE는 바로 이 "한글이나 워드 같은 존재"입니다. 메모장으로도 코드를 쓸 수는 있지만, IDE를 쓰면 백만 배 편해져요!

    정확한 정의

    IDE는 Integrated Development Environment의 약자로, 한국어로는 "통합 개발 환경"이라고 해요. 코드를 작성하고, 오류를 찾아주고, 실행까지 한 곳에서 할 수 있는 프로그램이에요.

    예시로 확인

    IDE가 제공하는 편리한 기능들을 살펴볼게요:

    • 코드 색상 표시: 키워드는 파란색, 문자열은 빨간색 등으로 자동으로 색칠해 줘서 코드를 읽기 쉬워져요
    • 자동 완성: 몇 글자만 입력하면 나머지를 제안해 줘요 (스마트폰 자동완성처럼!)
    • 오류 밑줄: 틀린 부분에 빨간 밑줄을 그어줘요 (워드의 맞춤법 검사처럼!)
    • 원클릭 실행: 버튼 하나로 코드를 실행할 수 있어요

    우리가 사용할 IDE는 Visual Studio Code(줄여서 VS Code)예요. 전 세계 개발자들이 가장 많이 사용하는 무료 IDE입니다.

    개념 2: 컴파일러 (Compiler) 🔄¶

    비유로 시작해요

    여러분이 한국어로 편지를 썼는데, 이 편지를 받을 친구는 영어만 이해한다고 해볼게요. 이 경우 번역가가 필요하겠죠?

    컴파일러는 바로 이 "번역가"예요. 우리가 C언어(사람의 언어에 가까운 것)로 작성한 코드를 컴퓨터가 이해할 수 있는 기계어(0과 1로 이루어진 것)로 번역해 주는 프로그램이에요.

    정확한 정의

    컴파일러(Compiler)란, 프로그래밍 언어로 작성한 소스 코드를 기계어로 변환하는 프로그램이에요. C언어 코드를 컴퓨터가 실행할 수 있는 파일(.exe)로 바꿔주는 역할을 합니다.

    예시로 확인

    flowchart LR A["📝 우리가 작성한 코드\n(C언어 - 사람이 읽을 수 있음)"] B["🔄 컴파일러\n(번역기)"] C["💻 실행 파일\n(기계어 - 컴퓨터가 이해함)"] A --> B --> C

    우리가 printf("Hello!");라고 쓰면, 컴파일러가 이것을 01001010 11010010... 같은 0과 1의 조합으로 번역해요. 컴퓨터는 이 0과 1을 읽고 "아, 화면에 Hello!를 출력하라는 뜻이구나!" 하고 실행하는 거예요.

    우리가 사용할 컴파일러는 GCC(GNU Compiler Collection)라는 것으로, 세계에서 가장 널리 쓰이는 무료 C언어 컴파일러입니다.

    💡 잠깐! "IDE와 컴파일러가 뭐가 다른 거지?" 라고 헷갈리시나요? 간단해요! IDE는 코드를 작성하는 편집기(글을 쓰는 워드), 컴파일러는 코드를 번역하는 번역기(번역가)예요. 둘 다 있어야 코딩을 할 수 있어요!

    개념 3: 소스 코드와 실행 파일 📄➡️⚙️¶

    비유로 시작해요

    악보와 음악의 관계를 생각해 보세요. 악보는 종이에 적힌 음표일 뿐이고, 이것을 피아니스트가 연주해야 비로소 "음악"이 됩니다.

    • 악보 = 소스 코드 (우리가 작성한 텍스트 파일)
    • 피아니스트 = 컴파일러 (번역하고 변환하는 역할)
    • 음악 = 실행 파일 (컴퓨터가 실제로 실행하는 프로그램)

    정확한 정의

    • 소스 코드(Source Code): 프로그래머가 프로그래밍 언어로 작성한 텍스트 파일. C언어의 경우 .c 확장자를 사용해요 (예: hello.c)
    • 실행 파일(Executable): 컴파일러가 소스 코드를 번역해서 만들어낸 파일. Windows에서는 .exe 확장자를 사용해요 (예: hello.exe)

    예시로 확인

    flowchart LR A["hello.c\n(소스 코드)"] -->|"컴파일"| B["hello.exe\n(실행 파일)"] B -->|"실행"| C["화면에 결과 출력!"]

    우리가 이번 장에서 할 일을 정리하면: hello.c 파일을 만들고 → 컴파일러로 번역하고 → hello.exe를 실행해서 → 화면에 결과를 확인하는 거예요!

    개념 4: 터미널 (Terminal) ⌨️¶

    비유로 시작해요

    음성 비서 시리(Siri)나 빅스비를 아시나요? "내일 날씨 알려줘"라고 말로 명령하면 결과를 알려주죠.

    터미널은 이것과 비슷해요. 다만, 말 대신 글자를 타이핑해서 컴퓨터에게 명령을 내리는 창이에요.

    정확한 정의

    터미널(Terminal)은 텍스트 명령어를 입력하여 컴퓨터를 조작하는 도구예요. "명령 프롬프트"나 "PowerShell"이라고도 불러요.

    예시로 확인

    터미널에서 이런 명령을 입력하면:

    gcc --version

    컴파일러가 제대로 설치되었다면 이런 결과가 나와요:

    gcc (Rev6, Built by MSYS2 project) 13.2.0

    이것은 "내 컴퓨터에 GCC 컴파일러 13.2.0 버전이 설치되어 있어요"라는 뜻이에요.

    💡 잠깐! 여기까지 이해가 잘 되셨나요? IDE, 컴파일러, 소스 코드, 터미널... 새로운 용어가 많이 나왔죠? 완벽하게 외울 필요는 없어요. 직접 설치하고 사용하다 보면 자연스럽게 익숙해질 거예요! 이제 진짜로 설치를 시작해 볼까요? 🚀

    🔨 따라하기: 개발 환경 설치¶

    자, 이제 실제로 여러분의 컴퓨터에 개발 환경을 설치해 볼게요!

    전체 과정은 아래와 같아요:

    flowchart TD S1["Step 1\nVS Code 설치"] S2["Step 2\nMSYS2로 GCC 컴파일러 설치"] S3["Step 3\n환경 변수 설정"] S4["Step 4\nVS Code에 C언어 확장 설치"] S5["Step 5\nHello World 코드 작성 및 실행"] S6["Step 6\n설치 확인 & 축하!"] S1 --> S2 --> S3 --> S4 --> S5 --> S6

    ⏱️ 시간 안내: Step 1~4는 설치 과정(약 40분), Step 5~6은 실습과 확인(약 20분)이에요. 다운로드 속도에 따라 시간이 다를 수 있으니 여유롭게 진행하세요!

    📌 참고: 이 가이드는 Windows 기준으로 작성되어 있어요. Mac 사용자는 Step 2에서 다른 방법을 안내해 드릴게요.

    Step 1: VS Code 설치하기 🛠️¶

    목표: 코드를 편하게 작성할 수 있는 편집기(IDE)를 설치해요.

    따라하기:

    1-1. 웹 브라우저를 열고 주소창에 다음을 입력하세요:

    https://code.visualstudio.com

    1-2. 홈페이지가 열리면 화면 중앙의 큰 파란색 버튼 "Download for Windows"를 클릭하세요.

    📌 Mac 사용자는 "Download for Mac" 버튼이 보여요. 그걸 클릭하면 됩니다!

    1-3. 다운로드가 완료되면 받은 파일(VSCodeUserSetup-x64-*.exe)을 더블클릭해서 실행하세요.

    1-4. 설치 마법사가 나타나면 다음 순서대로 진행하세요:

    • "동의합니다"에 체크 → 다음 클릭
    • 설치 경로는 기본값 그대로 → 다음 클릭
    • ⭐ 중요! "추가 작업 선택" 화면에서 다음 항목들에 모두 체크하세요:
    • ✅ "PATH에 추가(다시 시작한 후 사용 가능)" — 이것은 반드시!
    • ✅ "Code(으)로 열기" 작업을 추가 (파일/폴더 모두)
    • 설치 클릭 → 설치 완료 → 마침 클릭

    1-5. VS Code가 자동으로 실행되면 성공이에요! 🎉

    ✅ 체크포인트: VS Code가 열리면서 "Welcome" 탭이 보여야 해요. 어두운 배경에 영어로 된 환영 화면이 나타나면 정상이에요!

    Step 2: GCC 컴파일러 설치하기 🔧¶

    목표: C언어 코드를 번역해 줄 컴파일러를 설치해요.

    이 단계가 가장 중요하면서도 초보자가 어려워하는 부분이에요. 차근차근 따라오세요!

    Windows 사용자¶

    우리는 MSYS2라는 도구를 사용해서 GCC를 설치할 거예요.

    2-1. 웹 브라우저에서 다음 주소로 이동하세요:

    https://www.msys2.org

    2-2. 페이지 상단의 "Installation" 섹션에서 설치 파일 링크(예: msys2-x86_64-*.exe)를 클릭하여 다운로드하세요.

    2-3. 다운로드된 파일을 더블클릭하여 실행하세요.

    2-4. 설치 마법사가 나타나면:

    • 설치 경로는 기본값 C:\msys64 그대로 두세요 → 다음 클릭
    • 나머지도 기본값 그대로 → 다음 → 설치 클릭
    • 설치 완료 후 "MSYS2를 지금 실행" 체크 → 마침 클릭

    2-5. MSYS2 터미널 창(검은 배경)이 열리면, 다음 명령어를 정확히 입력하고 Enter를 누르세요:

    pacman -S mingw-w64-ucrt-x86_64-gcc

    2-6. 설치를 진행하시겠습니까? [Y/n] 라는 질문이 나오면 Y를 입력하고 Enter를 누르세요.

    2-7. 설치가 완료되면 잠시 기다렸다가 MSYS2 창을 닫아도 됩니다.

    💡 TIP: 명령어를 직접 타이핑하기 어려우면, 이 교재에서 명령어를 복사(Ctrl+C)한 뒤 MSYS2 창에서 마우스 오른쪽 클릭하면 붙여넣기가 됩니다!

    Mac 사용자¶

    Mac에서는 훨씬 간단해요!

    2-1. Spotlight(⌘ + Space)를 눌러 "Terminal"을 검색하고 실행하세요.

    2-2. 터미널에 다음 명령어를 입력하고 Enter:

    xcode-select --install

    2-3. 팝업 창이 뜨면 "설치"를 클릭하세요. (몇 분 소요)

    이것으로 Mac에서는 C 컴파일러 설치가 완료됩니다! 👏

    ✅ 체크포인트: 이 단계에서 설치가 정상적으로 끝나면, 터미널에 아무런 에러 메시지 없이 깔끔하게 완료되어야 해요. 빨간색 ERROR 메시지가 보이면 뒤의 "자주 하는 실수" 섹션을 확인해 주세요!

    Step 3: 환경 변수 설정하기 (Windows만 해당) ⚙️¶

    목표: 어디서든 컴파일러를 사용할 수 있도록 컴퓨터에 "길"을 알려줘요.

    📌 Mac 사용자는 이 단계를 건너뛰고 Step 4로 가세요!

    환경 변수가 뭔가요?

    비유하면 이래요. 친구에게 "카페 가자"라고 했을 때, 친구가 그 카페의 위치를 알고 있으면 바로 갈 수 있죠? 하지만 위치를 모르면 "어디 있는데?"라고 물어야 해요.

    환경 변수는 컴퓨터에게 "GCC 컴파일러는 여기에 있어!"라고 위치를 알려주는 설정이에요. 이걸 해놓으면 어느 폴더에서든 gcc 명령어를 사용할 수 있어요.

    따라하기:

    3-1. 키보드에서 Windows키 + S를 눌러 검색창을 열고, "환경 변수"라고 입력하세요.

    3-2. "시스템 환경 변수 편집"을 클릭하세요.

    3-3. 열린 창에서 오른쪽 아래의 "환경 변수(N)..." 버튼을 클릭하세요.

    3-4. 위쪽 "사용자 변수" 목록에서 Path를 찾아 클릭한 뒤, "편집" 버튼을 클릭하세요.

    3-5. "새로 만들기" 버튼을 클릭하고, 다음 경로를 입력하세요:

    C:\msys64\ucrt64\bin

    3-6. 확인 → 확인 → 확인을 눌러 모든 창을 닫으세요.

    3-7. 설정이 적용되려면 VS Code를 포함한 열려 있는 프로그램을 모두 닫고 다시 여세요.

    ⚠️ 주의! 3-5에서 경로를 입력할 때 오타가 있으면 컴파일러를 찾지 못해요. 특히 ucrt64를 ucrt나 ucrt46으로 잘못 입력하는 경우가 많으니 꼭 확인하세요!

    설치 확인하기 🔍

    환경 변수 설정이 제대로 되었는지 확인해 볼까요?

    3-8. Windows키 + S를 눌러 "cmd"를 검색하고, "명령 프롬프트"를 실행하세요.

    3-9. 다음 명령어를 입력하고 Enter:

    gcc --version

    3-10. 아래와 비슷한 메시지가 나오면 성공이에요! 🎉

    gcc.exe (Rev2, Built by MSYS2 project) 14.2.0
Copyright (C) 2024 Free Software Foundation, Inc.

    ✅ 체크포인트: gcc --version을 입력했을 때 버전 정보가 표시되면 성공! 만약 'gcc'은(는) 내부 또는 외부 명령...이 아닙니다라는 메시지가 나오면 환경 변수 설정을 다시 확인해 주세요.

    💡 잠깐! 여기까지 잘 따라오셨나요? 환경 변수 설정이 이 과정에서 가장 까다로운 부분이에요. 여기를 통과했다면 나머지는 정말 쉬워요! 거의 다 왔어요! 💪

    Step 4: VS Code에 C언어 확장 설치하기 🧩¶

    목표: VS Code가 C언어를 잘 이해하도록 "추가 기능"을 설치해요.

    VS Code는 기본적으로 다양한 언어를 지원하지만, 확장(Extension)을 설치하면 훨씬 더 편리하게 사용할 수 있어요. 스마트폰에 앱을 설치하는 것과 비슷해요!

    따라하기:

    4-1. VS Code를 실행하세요.

    4-2. 왼쪽 사이드바에서 네모 4개가 합쳐진 아이콘 (Extensions, 확장)을 클릭하세요. 또는 키보드에서 Ctrl + Shift + X를 누르세요.

    4-3. 검색창에 "C/C++"를 입력하세요.

    4-4. 검색 결과에서 "C/C++" (제작자: Microsoft)를 찾아 Install(설치) 버튼을 클릭하세요.

    📌 Microsoft가 만든 것을 설치해야 해요! 비슷한 이름의 다른 확장과 헷갈리지 마세요.

    4-5. 설치가 완료되면 "Install" 버튼이 "Uninstall"로 바뀌어요.

    추가로 설치하면 좋은 확장:

    검색창에 "Code Runner"를 입력하고 (제작자: Jun Han) 설치하세요. 이 확장을 설치하면 코드를 버튼 하나로 실행할 수 있어서 매우 편리해요!

    ✅ 체크포인트: 왼쪽 사이드바의 확장 아이콘에 설치된 확장이 보이고, "C/C++"와 "Code Runner"가 목록에 있으면 성공!

    Step 5: Hello, World! 코드 작성 및 실행하기 🌍¶

    목표: 첫 번째 C언어 프로그램을 작성하고 실행해요!

    드디어 가장 기다려왔던 순간이에요! 프로그래밍 역사에서 가장 유명한 프로그램, "Hello, World!"를 만들어 볼 거예요.

    따라하기:

    5-1. 먼저 코드를 저장할 폴더를 만들어요.

    • 바탕화면이나 원하는 위치에 "c_study"라는 이름의 폴더를 새로 만드세요.

    5-2. VS Code에서 파일(File) → 폴더 열기(Open Folder)를 클릭하고, 방금 만든 c_study 폴더를 선택하세요.

    📌 "이 폴더의 작성자를 신뢰합니까?"라는 팝업이 나오면 "예, 작성자를 신뢰합니다"를 클릭하세요.

    5-3. 왼쪽 탐색기 패널에서 새 파일 아이콘(종이에 + 표시)을 클릭하고, 파일 이름을 입력하세요:

    hello.c

    ⚠️ 반드시 .c로 끝나야 해요! hello.c이지 hello.txt가 아니에요. .c 확장자가 "이 파일은 C언어 코드입니다"라고 알려주는 거예요.

    5-4. 오른쪽 편집 영역에 아래 코드를 정확히 입력하세요:

    #include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

    5-5. Ctrl + S를 눌러 파일을 저장하세요.

    5-6. 코드를 실행해요! 두 가지 방법이 있어요:

    방법 A — Code Runner 사용 (가장 쉬움): - 오른쪽 위의 ▶ (재생) 버튼을 클릭하세요. - 또는 Ctrl + Alt + N을 누르세요.

    방법 B — 터미널에서 직접 실행: - VS Code 상단 메뉴에서 터미널(Terminal) → **새 터미널
