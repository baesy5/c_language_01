    Chapter 09. 왜 코드를 나눠야 할까? — 함수가 필요한 이유¶

    Part 3: 코드를 깔끔하게 정리하는 힘 — 함수

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • 코드가 길어질 때 발생하는 구체적인 문제점 3가지를 설명할 수 있다
    • 함수의 개념을 일상 속 비유(자판기, 레시피)로 설명할 수 있다
    • printf, scanf가 누군가 미리 만들어둔 함수임을 인식할 수 있다
    • 왜 직접 함수를 만들어야 하는지 그 동기를 자신의 말로 설명할 수 있다

    ⏱️ 예상 학습 시간: 약 1시간 30분

    📚 미리 알아야 할 것¶

    이전 장들에서 우리는 꽤 많은 것을 배웠어요! 간단히 복습해 볼까요?

    • 변수와 자료형 (Chapter 03~04): 데이터를 담는 상자, int, float, char 등
    • 조건문 (Chapter 05~06): if, else, switch로 흐름을 분기하기
    • 반복문 (Chapter 07~08): for, while로 같은 작업을 반복하기
    • printf와 scanf: 화면에 출력하고, 키보드로 입력받기

    특히 Chapter 08에서 만들었던 숫자 맞추기 게임을 기억하시나요? 그 코드를 다시 꺼내볼 거예요. 코드가 점점 길어지면서 느꼈던 "뭔가 복잡하다"는 느낌… 오늘 그 느낌의 정체를 알게 됩니다. 😊

    💡 왜 이걸 배우나요?¶

    여러분, 혹시 이런 경험 있으신가요?

    "코드가 잘 돌아가긴 하는데… 어디가 뭔지 모르겠어요." "비슷한 코드를 또 써야 하는데, 복사해서 붙여넣기 해도 되나요?" "오류가 났는데, 100줄 넘는 코드에서 어디가 문제인지 찾기 힘들어요."

    프로그래밍을 배우다 보면, 어느 순간 코드가 급격히 길어지는 시점이 옵니다. 변수 선언, 입력 처리, 계산, 출력, 반복… 이 모든 것이 main 함수 하나에 몰려 있으면 마치 모든 짐을 하나의 가방에 욱여넣은 것과 같아요.

    오늘 배울 함수(Function)는 이 문제를 해결하는 가장 기본적이고 강력한 도구입니다. 함수를 배우면:

    • 🧹 코드가 깔끔하게 정리됩니다
    • 🔄 같은 코드를 여러 번 재사용할 수 있습니다
    • 🐛 버그를 쉽게 찾고 고칠 수 있습니다

    이번 장은 함수를 직접 만드는 것보다, "왜 함수가 필요한가"를 마음 깊이 이해하는 것이 목표입니다. 동기 부여가 확실하면, 다음 장에서 함수를 만드는 건 훨씬 쉬워져요!

    📚 핵심 개념¶

    개념 1: 코드가 길어지면 생기는 문제들 🤯¶

    1) 비유로 시작¶

    이사를 한다고 상상해 보세요. 짐을 정리할 때 두 가지 방법이 있습니다.

    방법 A — 상자 하나에 다 넣기: 옷, 그릇, 책, 화장품, 충전기, 식재료… 전부 하나의 커다란 상자에 넣습니다.

    방법 B — 종류별로 나눠 넣기: "옷 상자", "주방용품 상자", "책 상자", "전자기기 상자"로 나눠서 라벨을 붙입니다.

    방법 A로 이사하면 어떻게 될까요? 짐을 풀 때 숟가락 하나 찾으려고 상자 전체를 뒤져야 해요. 깨지기 쉬운 그릇이 책에 눌려 깨질 수도 있고요. 😰

    프로그래밍도 똑같습니다. main 함수 하나에 모든 코드를 넣는 건 방법 A와 같아요.

    2) 구체적인 문제 3가지¶

    지금까지 배운 내용을 활용해서, 간단한 성적 처리 프로그램을 생각해 봅시다. main 안에 모든 걸 넣으면 이런 식이 됩니다:

    #include <stdio.h>

int main(void) {
    // --- 학생 3명의 점수 입력 ---
    int score1, score2, score3;
    printf("학생 1의 점수: ");
    scanf("%d", &score1);
    printf("학생 2의 점수: ");
    scanf("%d", &score2);
    printf("학생 3의 점수: ");
    scanf("%d", &score3);

    // --- 평균 계산 ---
    float average = (score1 + score2 + score3) / 3.0;

    // --- 등급 판정 ---
    char grade;
    if (average >= 90) {
        grade = 'A';
    } else if (average >= 80) {
        grade = 'B';
    } else if (average >= 70) {
        grade = 'C';
    } else if (average >= 60) {
        grade = 'D';
    } else {
        grade = 'F';
    }

    // --- 결과 출력 ---
    printf("\n=== 성적 결과 ===\n");
    printf("평균: %.1f\n", average);
    printf("등급: %c\n", grade);

    // --- 격려 메시지 ---
    if (grade == 'A') {
        printf("훌륭합니다!\n");
    } else if (grade == 'B') {
        printf("잘 했어요!\n");
    } else if (grade == 'C') {
        printf("조금 더 노력해봐요!\n");
    } else {
        printf("힘내세요!\n");
    }

    return 0;
}

    이 코드는 약 40줄이에요. 아직 괜찮아 보이죠? 하지만 현실에서는 이런 요구사항이 추가됩니다:

    • "학생이 3명이 아니라 30명이면요?"
    • "과목이 국어, 영어, 수학 3개면요?"
    • "과목별로 등급도 따로 매겨야 하면요?"

    이러면 코드가 순식간에 200줄, 300줄로 늘어납니다. 그리고 세 가지 심각한 문제가 발생해요:

    문제 ① — 가독성이 떨어진다 📖

    200줄짜리 코드에서 "등급 판정하는 부분이 어디지?"라고 찾으려면, 위에서 아래로 쭉 스크롤하면서 눈으로 훑어야 합니다. 주석을 달아도, 코드가 길어지면 찾기가 힘들어요.

    문제 ② — 같은 코드를 반복 작성한다 🔄

    등급 판정 로직이 국어, 영어, 수학 각각에 필요하다면? 같은 if-else 구조를 3번 복사해서 붙여넣어야 합니다. 만약 등급 기준이 바뀌면(예: 95점 이상이 A+)? 3곳 모두 수정해야 해요. 하나라도 빠뜨리면 버그!

    문제 ③ — 오류 수정이 어렵다 🐛

    코드 전체가 하나의 덩어리이기 때문에, 어디서 문제가 생겼는지 범위를 좁히기 어렵습니다. "평균이 이상하게 나와요"라는 문제가 생기면, 입력 부분이 잘못된 건지, 계산 부분이 잘못된 건지, 출력 부분이 잘못된 건지 하나하나 확인해야 해요.

    3) 정리¶

    코드가 길어졌을 때 발생하는 문제를 정리하면:

    • 가독성 저하: 코드를 읽기 어려워진다
    • 코드 중복: 같은 로직을 여러 번 복사/붙여넣기 한다
    • 유지보수 곤란: 수정할 곳을 찾기 힘들고, 하나를 고치면 다른 곳도 고쳐야 한다

    💡 잠깐! 이 세 가지 문제, 공감이 되시나요? 아직 짧은 코드만 작성해 봤다면 "나는 아직 괜찮은데?"라고 느낄 수 있어요. 하지만 이번 장을 끝까지 따라오시면, 미리 좋은 습관을 들이는 것이 얼마나 중요한지 느끼실 거예요!

    개념 2: 함수란 무엇인가? — 자판기 비유 🥤¶

    1) 비유로 시작¶

    자판기를 떠올려 보세요.

    여러분이 자판기 앞에서 하는 일은 딱 세 가지입니다:

    1. 동전을 넣는다 (입력)
    2. 버튼을 누른다 (실행 요청)
    3. 음료가 나온다 (출력)

    여러분은 자판기 내부에서 무슨 일이 일어나는지 알 필요가 없어요. 동전이 어떤 경로로 떨어지는지, 음료가 어떤 메커니즘으로 나오는지 몰라도 됩니다. 그냥 "돈을 넣고 버튼을 누르면 음료가 나온다"는 것만 알면 돼요.

    flowchart LR A["🪙 동전 넣기\n(입력)"] --> B["🤖 자판기 내부 처리\n(처리)"] B --> C["🥤 음료 나옴\n(출력)"] style A fill:#FFE0B2,stroke:#F57C00,color:#000 style B fill:#E3F2FD,stroke:#1976D2,color:#000 style C fill:#C8E6C9,stroke:#388E3C,color:#000

    프로그래밍의 함수도 정확히 이렇게 작동합니다!

    • 입력: 함수에게 전달하는 데이터 (동전 역할)
    • 처리: 함수 내부에서 일어나는 작업 (자판기 내부 메커니즘)
    • 출력: 함수가 돌려주는 결과 (음료 역할)

    2) 정확한 정의¶

    📌 함수(Function)란, 특정한 작업을 수행하는 코드 묶음에 이름을 붙인 것입니다.

    필요할 때마다 그 이름을 불러서(호출해서) 사용할 수 있습니다.

    핵심 키워드를 하나씩 뜯어볼게요:

    • "특정한 작업": 함수 하나는 한 가지 일을 담당합니다. "평균 계산", "등급 판정", "결과 출력" 각각이 하나의 함수가 될 수 있어요.
    • "코드 묶음": 여러 줄의 코드를 하나로 묶어놓은 것입니다.
    • "이름을 붙인 것": calculate_average, get_grade 같은 이름을 붙여서, 나중에 이름만으로 불러서 사용할 수 있어요.

    3) 레시피 비유로 확장 🧑‍🍳¶

    자판기 비유를 하나 더 발전시켜 볼게요. 요리 레시피를 생각해 보세요.

    여러분이 친구에게 김치찌개 만드는 법을 알려준다고 합시다:

    "먼저 밥을 지어. 그리고 김치찌개를 끓여. 마지막으로 상을 차려."

    이 세 문장만으로 충분하죠? 하지만 "밥을 지어"라는 말 안에는 사실 이런 과정이 숨어 있어요:

    쌀을 씻는다 → 물을 붓는다 → 30분 불린다 → 밥솥에 넣는다 → 취사 버튼을 누른다 → 15분 기다린다 → 뜸 들인다

    이 세부 과정을 "밥짓기"라는 이름으로 묶어둔 거예요. 매번 "쌀을 씻고 물을 붓고 30분 불리고…"라고 말하지 않아도, "밥을 지어" 한마디면 되는 거죠.

    함수도 마찬가지입니다:

    • "밥을 지어" → cook_rice() 함수
    • "김치찌개를 끓여" → make_stew() 함수
    • "상을 차려" → set_table() 함수

    전체 요리 과정은 이렇게 깔끔해집니다:

    cook_rice();       // 밥을 짓고
make_stew();       // 김치찌개를 끓이고
set_table();       // 상을 차린다

    세부 과정은 각 함수 안에 숨어 있으니까, 전체 흐름을 파악하기가 훨씬 쉽죠!

    개념 3: 우리는 이미 함수를 쓰고 있었다! 🤯¶

    1) 비유로 시작¶

    여러분이 스마트폰을 처음 샀을 때를 떠올려 보세요. 카메라 앱, 전화 앱, 문자 앱이 이미 설치되어 있었죠? 여러분이 직접 만들지 않았지만, 누군가 미리 만들어 놓은 앱을 그냥 사용하고 있었어요.

    C언어의 printf와 scanf도 마찬가지입니다!

    2) 정확한 이해¶

    첫 번째 프로그램부터 우리가 써왔던 코드를 다시 볼게요:

    printf("안녕하세요!\n");
scanf("%d", &number);

    이 printf와 scanf는 사실 함수입니다! C언어를 만든 사람들이 미리 만들어 놓은 함수예요.

    • printf 함수: "화면에 글자를 출력하라"는 복잡한 작업을 하나의 이름으로 묶어놓은 것
    • scanf 함수: "키보드에서 입력을 받아라"는 복잡한 작업을 하나의 이름으로 묶어놓은 것

    우리가 맨 위에 쓰던 #include <stdio.h>의 의미가 이제 좀 더 명확해집니다:

    📌 #include <stdio.h>: "표준 입출력(Standard I/O) 관련 함수들의 설명서를 가져와라"

    이 한 줄 덕분에 printf, scanf 같은 미리 만들어진 함수를 사용할 수 있었던 거예요!

    화면에 "Hello"를 출력하려면, 컴퓨터 내부에서는 실제로 매우 복잡한 일이 벌어집니다. 문자를 하나씩 변환하고, 운영체제에 요청하고, 화면 버퍼에 쓰고… 하지만 우리는 그 복잡한 과정을 전혀 몰라도 printf("Hello")라고 쓰면 되죠!

    이것이 함수의 힘입니다.

    3) 지금까지 사용한 함수들 되돌아보기¶

    지금까지 우리가 써왔던 것 중에서 함수인 것들을 정리해 볼게요:

    • printf() — 화면에 출력하는 함수
    • scanf() — 키보드로 입력받는 함수
    • rand() — 랜덤 숫자를 만드는 함수 (숫자 맞추기 게임에서 사용!)
    • srand() — 랜덤 시드를 설정하는 함수
    • time() — 현재 시간을 가져오는 함수
    • main() — 네, 이것도 함수입니다! 프로그램이 시작되면 가장 먼저 실행되는 특별한 함수예요.

    🎉 아하! 순간

    "이미 함수를 쓰고 있었구나!"

    맞아요! 여러분은 이미 함수의 사용자였어요. 이제부터는 함수의 제작자가 되는 겁니다. 누군가 printf를 만들어서 우리가 편하게 쓴 것처럼, 우리도 편리한 함수를 만들 수 있어요!

    개념 4: 함수를 쓰면 뭐가 좋을까? — 함수의 3대 장점 ✨¶

    앞에서 코드가 길어지면 생기는 문제 3가지를 봤죠? 함수는 그 문제를 정확히 해결합니다.

    1) 비유로 시작 — 회사 조직도¶

    작은 가게를 운영한다고 생각해 보세요.

    사장 혼자 다 하는 가게: 주문 받기, 요리하기, 서빙하기, 계산하기, 청소하기… 한 사람이 전부 합니다. 가게가 작을 때는 괜찮지만, 손님이 많아지면? 😵

    직원이 나뉜 가게: 캐셔는 계산만, 요리사는 요리만, 서빙은 홀 직원이… 각자 맡은 일만 하니까 효율적이죠!

    main 함수 하나에 모든 코드를 넣는 건 사장 혼자 다 하는 가게이고, 함수를 나누는 건 전문 직원을 고용하는 것입니다.

    2) 장점 정리¶

    장점 ① — 가독성: 코드가 읽기 쉬워진다 📖

    함수를 사용하면, main 함수가 이렇게 바뀝니다:

    // Before: main 안에 모든 게 뒤섞임 (40줄 이상)
int main(void) {
    // 입력 코드 10줄...
    // 계산 코드 5줄...
    // 등급 판정 코드 15줄...
    // 출력 코드 10줄...
    // 격려 메시지 코드 10줄...
}

    // After: main이 깔끔한 목차처럼 됨
int main(void) {
    int score = get_score();        // 점수 입력
    float avg = calculate_average(); // 평균 계산
    char grade = get_grade(avg);    // 등급 판정
    print_result(avg, grade);       // 결과 출력
    return 0;
}

    아래쪽 코드가 마치 책의 목차처럼 읽히지 않나요? "점수 입력 → 평균 계산 → 등급 판정 → 결과 출력" 흐름이 한눈에 보입니다!

    장점 ② — 재사용: 한 번 만들면 여러 번 쓴다 🔄

    등급 판정 함수 get_grade를 한 번 만들어 두면:

    char korean_grade = get_grade(korean_avg);   // 국어 등급
char english_grade = get_grade(english_avg); // 영어 등급
char math_grade = get_grade(math_avg);       // 수학 등급

    같은 함수를 3번 호출하면 끝! 같은 코드를 3번 복사/붙여넣기할 필요가 없어요.

    장점 ③ — 유지보수: 고칠 곳이 한 곳이다 🔧

    등급 기준이 바뀌면? (예: 95점 이상이 A+로 변경)

    • 함수 없이: 코드에서 등급 판정 부분을 모두 찾아서 각각 수정 (3곳? 5곳? 10곳?)
    • 함수 있으면: get_grade 함수 하나만 수정하면 끝!
