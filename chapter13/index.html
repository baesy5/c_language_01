    데이터를 줄 세우기 — 배열 기초¶

    Part 4: [2권] 게임을 더 풍성하게 — 배열과 실전 도구

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • 배열의 개념을 이해하고, 같은 자료형의 데이터 여러 개를 하나의 변수로 저장할 수 있다
    • 배열을 선언하고 초기화하며, 인덱스를 통해 원하는 데이터에 접근할 수 있다
    • 반복문과 배열을 결합하여 여러 데이터를 효율적으로 처리할 수 있다
    • 배열 범위를 벗어나는 접근이 왜 위험한지 설명할 수 있다

    예상 학습 시간: 약 2시간 ⏱️

    📚 미리 알아야 할 것¶

    이전 장에서 배운 다음 내용이 필요해요:

    • 변수 선언과 자료형: int score = 100;처럼 변수에 값을 저장하는 방법
    • for 반복문: for (int i = 0; i < 5; i++)처럼 정해진 횟수만큼 반복하는 방법
    • scanf와 printf: 키보드 입력을 받고 화면에 출력하는 방법

    혹시 기억이 가물가물하다면, 이전 장을 가볍게 훑어보고 오세요! 😊

    💡 왜 이걸 배우나요?¶

    게임을 만든다고 상상해 보세요. 5명의 플레이어 점수를 저장하려면 어떻게 해야 할까요?

    지금까지 배운 방법으로는 이렇게 해야 합니다:

    int score1 = 85;
int score2 = 92;
int score3 = 78;
int score4 = 96;
int score5 = 88;

    5명이니까 그럭저럭 할 만하죠? 그런데 100명이라면요? 변수 100개를 일일이 만들어야 할까요? 😱

    배열(Array) 을 사용하면 이 문제가 단번에 해결됩니다:

    int scores[100];  // 끝! 100명분의 공간이 한 줄로 준비됨

    배열은 게임 속 아이템 목록, 학생 성적표, 로또 번호 저장 등 같은 종류의 데이터를 여러 개 다뤄야 하는 모든 상황에서 핵심적으로 사용됩니다. 프로그래밍에서 가장 기본이 되는 자료구조이니, 이번 장에서 확실히 익혀 봅시다! 💪

    📚 핵심 개념¶

    개념 1: 배열이란? — 일렬로 늘어선 사물함 🗄️¶

    비유로 시작해요

    학교 복도에 사물함이 일렬로 쭉 늘어서 있는 모습을 떠올려 보세요. 모든 사물함은 같은 크기이고, 각 사물함에는 번호가 붙어 있어요. 배열은 바로 이 사물함과 똑같습니다!

    graph LR subgraph "int scores[5] — 정수 5칸짜리 배열" A["[0]\n85"] --- B["[1]\n92"] --- C["[2]\n78"] --- D["[3]\n96"] --- E["[4]\n88"] end

    정확한 정의

    🔑 배열(Array) 이란 같은 자료형의 데이터를 연속된 메모리 공간에 여러 개 저장하는 자료구조입니다.

    핵심 특징을 정리하면:

    • 같은 자료형만: int 배열에는 정수만, float 배열에는 실수만 들어가요
    • 크기가 고정: 처음 만들 때 칸 수를 정하면 나중에 바꿀 수 없어요
    • 연속 저장: 메모리에서 칸들이 바로 옆에 나란히 붙어 있어요

    예시로 확인

    int scores[5];      // 정수 5개를 저장할 수 있는 배열
float temps[7];     // 실수 7개를 저장할 수 있는 배열 (일주일 온도 등)
char name[20];      // 문자 20개를 저장할 수 있는 배열 (이름 등)

    개념 2: 선언과 초기화 — 사물함 만들고 물건 넣기 📦¶

    비유로 시작해요

    사물함을 설치하는 것이 선언, 사물함에 물건을 넣는 것이 초기화입니다. 사물함을 설치만 해놓고 비워두면 안에 뭐가 들었는지 알 수 없겠죠? (쓰레기 값이 들어있을 수 있어요!)

    정확한 정의

    배열을 만드는 기본 문법은 다음과 같습니다:

    자료형 배열이름[크기];

    초기화 방법은 여러 가지가 있어요:

    // 방법 1: 선언과 동시에 값 넣기
int scores[5] = {85, 92, 78, 96, 88};

// 방법 2: 일부만 넣기 (나머지는 0으로 채워짐)
int scores[5] = {85, 92};  // → {85, 92, 0, 0, 0}

// 방법 3: 전부 0으로 초기화
int scores[5] = {0};       // → {0, 0, 0, 0, 0}

// 방법 4: 크기를 생략하면 값 개수로 자동 결정
int scores[] = {85, 92, 78, 96, 88};  // 크기 5로 자동 결정

    예시로 확인

    #include <stdio.h>

int main(void) {
    int scores[5] = {85, 92, 78, 96, 88};

    printf("첫 번째 점수: %d\n", scores[0]);  // 85 출력
    printf("세 번째 점수: %d\n", scores[2]);  // 78 출력

    return 0;
}

    💡 잠깐! scores[0]이 첫 번째라는 게 이상하게 느껴지시나요? 바로 다음 개념에서 자세히 설명할게요!

    개념 3: 인덱스 — 0번부터 시작하는 사물함 번호 🔢¶

    비유로 시작해요

    엘리베이터를 생각해 보세요. 한국에서는 1층부터 시작하지만, 어떤 나라에서는 0층(Ground Floor) 부터 시작하죠? C 언어의 배열은 바로 이 0층부터 시작하는 엘리베이터와 같습니다!

    정확한 정의

    🔑 인덱스(Index) 란 배열에서 각 요소의 위치를 나타내는 번호로, 항상 0부터 시작합니다.

    크기가 5인 배열이면 사용할 수 있는 인덱스는 0, 1, 2, 3, 4 — 총 다섯 개입니다.

    graph TB subgraph "scores 배열 — 크기 5" direction LR I0["인덱스 0<br/>scores[0]<br/>= 85"] I1["인덱스 1<br/>scores[1]<br/>= 92"] I2["인덱스 2<br/>scores[2]<br/>= 78"] I3["인덱스 3<br/>scores[3]<br/>= 96"] I4["인덱스 4<br/>scores[4]<br/>= 88"] end N1["❌ scores[5] → 존재하지 않음!"] style N1 fill:#ffcccc,stroke:#ff0000

    예시로 확인 — 값 읽기와 쓰기

    int scores[5] = {85, 92, 78, 96, 88};

// 읽기: 인덱스로 값을 꺼내기
printf("%d\n", scores[0]);   // 85

// 쓰기: 인덱스로 값을 바꾸기
scores[2] = 100;             // 78이었던 값이 100으로 변경!
printf("%d\n", scores[2]);   // 100

    🧠 꼭 기억하세요! 크기가 N인 배열의 마지막 인덱스는 N-1입니다. scores[5]는 이 배열에 존재하지 않아요!

    개념 4: 반복문 + 배열 — 최강 콤비 🤝¶

    비유로 시작해요

    사물함 100개를 하나하나 열어서 확인하는 건 힘들지만, 로봇이 0번부터 99번까지 자동으로 순서대로 열어준다면? for문이 바로 그 로봇입니다!

    정확한 정의

    배열의 인덱스는 0부터 순서대로 증가하고, for문의 변수 i도 0부터 순서대로 증가하니까, 이 둘은 천생연분입니다:

    for (int i = 0; i < 배열크기; i++) {
    // scores[i]로 각 요소에 접근
}

    예시로 확인

    int scores[5] = {85, 92, 78, 96, 88};

// 모든 점수 출력하기
for (int i = 0; i < 5; i++) {
    printf("scores[%d] = %d\n", i, scores[i]);
}

    실행 결과: 

    scores[0] = 85
scores[1] = 92
scores[2] = 78
scores[3] = 96
scores[4] = 88

    💡 잠깐! 여기까지 이해가 잘 되셨나요? 배열 선언 → 인덱스로 접근 → 반복문으로 순회, 이 세 가지가 배열의 핵심이에요. 어렵게 느껴져도 괜찮아요. 이제 직접 코드를 쳐보면 훨씬 와닿을 거예요! 😊

    🔨 따라하기 — 5명의 점수를 입력받아 평균 구하기¶

    🎯 실습 목표¶

    키보드로 5명의 점수를 입력받고, 전체 평균을 계산해서 출력하는 프로그램을 만들어 봅시다!

    Step 1: 배열 선언하기¶

    먼저 빈 배열을 선언합니다. 새 파일 score_avg.c를 만들고 아래 코드를 입력하세요.

    #include <stdio.h>

int main(void) {
    int scores[5];  // 5명의 점수를 저장할 배열

    printf("점수 평균 계산기\n");
    printf("================\n");

    return 0;
}

    ✅ 체크포인트: 컴파일하고 실행하면 제목만 출력되면 성공이에요!

    Step 2: 반복문으로 점수 입력받기¶

    배열에 값을 채워 넣겠습니다. return 0; 바로 위에 다음 코드를 추가하세요:

        // 5명의 점수 입력
    for (int i = 0; i < 5; i++) {
        printf("%d번째 학생 점수 입력: ", i + 1);
        scanf("%d", &scores[i]);
    }

    i + 1을 출력하는 이유를 알겠나요? 사용자에게 보여줄 때는 "1번째, 2번째…"가 자연스러우니까요. 하지만 실제 배열 인덱스는 i(0부터)를 사용합니다.

    ✅ 체크포인트: 실행하면 "1번째 학생 점수 입력:"부터 "5번째 학생 점수 입력:"까지 차례로 나오면 성공!

    Step 3: 합계 구하기¶

    입력받은 점수들을 모두 더합니다. 입력 코드 아래에 추가하세요:

        // 합계 계산
    int sum = 0;
    for (int i = 0; i < 5; i++) {
        sum += scores[i];  // sum = sum + scores[i]
    }

    Step 4: 평균 계산 및 출력¶

    합계를 인원수로 나눠서 평균을 구합니다:

        // 평균 계산 및 출력
    float average = (float)sum / 5;

    printf("\n--- 결과 ---\n");
    printf("합계: %d\n", sum);
    printf("평균: %.1f\n", average);

    (float)sum은 정수인 sum을 실수로 바꾸는 형변환이에요. 이렇게 해야 소수점 이하까지 정확하게 나옵니다. (예: 431 / 5 = 86이 아니라 86.2)

    Step 5: 최고 점수 찾기¶

    배열의 강력함을 체감하기 위해 최고 점수도 찾아봅시다:

        // 최고 점수 찾기
    int max = scores[0];  // 일단 첫 번째를 최고로 가정
    for (int i = 1; i < 5; i++) {
        if (scores[i] > max) {
            max = scores[i];  // 더 큰 값 발견 시 교체
        }
    }
    printf("최고 점수: %d\n", max);

    🧠 왜 i = 1부터 시작하나요? scores[0]은 이미 max에 넣었으니까, 비교는 1번부터 하면 충분해요!

    Step 6: 전체 점수 목록 출력¶

    마지막으로 입력한 점수를 한눈에 보여주는 코드를 추가합니다:

        // 전체 점수 목록
    printf("\n[전체 점수]\n");
    for (int i = 0; i < 5; i++) {
        printf("  %d번 학생: %d점\n", i + 1, scores[i]);
    }

    ✅ 체크포인트: 5명의 점수를 입력하면 합계, 평균, 최고 점수, 전체 목록이 모두 출력되면 대성공! 🎉

    📝 전체 코드¶

    #include <stdio.h>

int main(void) {
    int scores[5];

    printf("점수 평균 계산기\n");
    printf("================\n");

    // 5명의 점수 입력
    for (int i = 0; i < 5; i++) {
        printf("%d번째 학생 점수 입력: ", i + 1);
        scanf("%d", &scores[i]);
    }

    // 합계 계산
    int sum = 0;
    for (int i = 0; i < 5; i++) {
        sum += scores[i];
    }

    // 평균 계산
    float average = (float)sum / 5;

    // 최고 점수 찾기
    int max = scores[0];
    for (int i = 1; i < 5; i++) {
        if (scores[i] > max) {
            max = scores[i];
        }
    }

    // 결과 출력
    printf("\n--- 결과 ---\n");
    printf("합계: %d\n", sum);
    printf("평균: %.1f\n", average);
    printf("최고 점수: %d\n", max);

    printf("\n[전체 점수]\n");
    for (int i = 0; i < 5; i++) {
        printf("  %d번 학생: %d점\n", i + 1, scores[i]);
    }

    return 0;
}

    ⚠️ 자주 하는 실수¶

    실수 1: 인덱스를 1부터 시작하기 🔴¶

    int scores[5] = {85, 92, 78, 96, 88};

// ❌ 잘못된 코드
for (int i = 1; i <= 5; i++) {
    printf("%d\n", scores[i]);  // scores[5]는 존재하지 않음!
}

// ✅ 올바른 코드
for (int i = 0; i < 5; i++) {
    printf("%d\n", scores[i]);
}
    • 왜 이런 일이 생기나요? 일상에서는 "1번부터" 세는 게 자연스러워서 습관적으로 1부터 시작하게 돼요.
    • 이렇게 기억해요: C 배열은 0부터 시작, 조건은 < 크기. 이 패턴을 외우세요: for (int i = 0; i < N; i++)

    실수 2: 배열 범위를 벗어나는 접근 🔴¶

    int scores[5] = {85, 92, 78, 96, 88};

scores[5] = 100;   // ❌ 위험! 인덱스 5는 존재하지 않음
scores[-1] = 50;   // ❌ 위험! 음수 인덱스도 안 됨
    • 왜 이런 일이 생기나요? "5칸이니까 5번까지 있겠지"라고 생각하기 쉬워요. 하지만 0, 1, 2, 3, 4로 5칸이에요!
    • 이렇게 해결해요: C 언어는 범위를 벗어나도 오류 메시지 없이 실행되는 경우가 있어서 더 위험해요. 엉뚱한 메모리를 건드려서 프로그램이 갑자기 죽거나, 이상한 결과가 나올 수 있습니다. 마지막 인덱스 = 크기 - 1을 항상 확인하세요!

    • 이렇게 해결해요: 입문 단계에서는 배열 크기를 숫자 또는 #define 으로 고정하는 것이 안전합니다. 동적으로 크기를 바꾸는 방법은 나중에 포인터와 함께 배울 거예요.
