    직접 만드는 가위바위보 게임 — 코딩 실전¶

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • 설계한 함수 구조대로 가위바위보 게임을 완성할 수 있다
    • 컴퓨터의 랜덤 선택, 승패 판정, 전적 기록 기능을 함수로 구현할 수 있다
    • 반복 플레이와 최종 결과 출력 기능을 추가하여 완성도를 높일 수 있다
    • 코드를 점진적으로 작성하고 테스트하는 개발 방식을 체험할 수 있다

    ⏱️ 예상 학습 시간: 약 2시간 30분

    📚 미리 알아야 할 것¶

    이전 장(16장)에서 우리는 가위바위보 게임의 설계도를 만들었습니다. 혹시 기억이 잘 안 나시더라도 걱정 마세요. 핵심만 빠르게 복습해 볼게요.

    이전 장에서 준비한 것들:

    • 🎯 게임 흐름도: 사용자 입력 → 컴퓨터 선택 → 승패 판정 → 결과 출력 → 반복 여부 → 전적 출력
    • 🧩 함수 분리 계획: 기능별로 함수를 나누기로 했어요
    • 📦 변수/배열 설계: 전적을 기록할 변수들을 미리 정했어요

    이번 장에서는 그 설계도를 진짜 동작하는 코드로 바꿉니다. 마치 건축 설계도를 보고 집을 짓는 것처럼, 한 벽돌씩 쌓아올리겠습니다! 🧱

    💡 왜 이걸 배우나요?¶

    프로그래밍을 배우는 과정에서 가장 큰 벽은 "개별 문법은 아는데, 이걸 합쳐서 뭔가를 만들 수가 없어요" 라는 순간입니다.

    이번 장이 바로 그 벽을 넘는 경험입니다.

    지금까지 배운 변수, 조건문, 반복문, 함수, 배열 — 이 모든 것이 하나의 게임 안에서 어우러집니다. 조각들을 맞추는 퍼즐처럼, 각 문법이 제 자리를 찾아가는 걸 직접 느끼게 될 거예요.

    그리고 무엇보다, "내가 직접 게임을 만들었다!" 라는 성취감은 앞으로의 학습에 엄청난 동력이 됩니다. 🔥

    📚 핵심 개념¶

    이번 장에서 집중할 핵심 개념은 4가지입니다.

    개념 1: 점진적 개발 (Incremental Development)¶

    🎨 비유로 시작

    요리를 할 때, 모든 재료를 한꺼번에 냄비에 넣고 "맛있어져라!" 하지 않잖아요? 양파를 먼저 볶고 맛을 보고, 고기를 넣고 맛을 보고, 소스를 넣고 또 맛을 봅니다. 코딩도 똑같아요!

    📖 정확한 정의

    점진적 개발이란, 프로그램을 한 번에 완성하지 않고, 작은 부분을 만들 때마다 실행해서 확인하고, 제대로 되면 다음 부분을 추가하는 개발 방법입니다.

    💻 예시로 확인

    [나쁜 방법] 전체 코드 200줄을 한 번에 쓰고 → 실행 → 에러 폭탄 💣 → 어디가 문제인지 모름

[좋은 방법] 함수 1개 작성 → 실행 → 확인 ✅ → 함수 1개 추가 → 실행 → 확인 ✅ → ...

    이번 장에서는 이 좋은 방법을 직접 체험합니다. 매 단계마다 "여기까지 실행해 보세요!" 라고 안내할 거예요.

    개념 2: Math.random()을 활용한 컴퓨터 선택¶

    🎨 비유로 시작

    주사위를 던져서 1이 나오면 가위, 2가 나오면 바위, 3이 나오면 보를 낸다고 상상해 보세요. 컴퓨터에게 주사위를 쥐어주는 것이 Math.random()입니다.

    📖 정확한 정의

    Math.random()은 0 이상 1 미만의 랜덤 소수를 돌려주는 자바스크립트 내장 함수입니다. 이것을 가공하면 원하는 범위의 정수를 얻을 수 있어요.

    💻 예시로 확인

    // 0, 1, 2 중 하나를 랜덤으로 얻는 방법
let randomNumber = Math.floor(Math.random() * 3);
// Math.random() * 3  → 0 이상 3 미만의 소수 (예: 1.742)
// Math.floor(...)     → 소수점 아래를 버림 (예: 1)
// 결과: 0, 1, 2 중 하나

// 이것을 가위/바위/보에 매핑
let choices = ["가위", "바위", "보"];
let computerChoice = choices[randomNumber];
console.log("컴퓨터: " + computerChoice);

    💡 Math.floor()는 소수점 아래를 내림하는 함수예요. Math.floor(2.9)는 2가 됩니다.

    개념 3: 승패 판정 로직¶

    🎨 비유로 시작

    심판이 경기 결과를 판정하려면 규칙표가 필요하죠? 가위바위보의 규칙표는 우리 모두가 알고 있어요:

    • 가위는 보를 이긴다 ✂️ > 📄
    • 바위는 가위를 이긴다 ✊ > ✂️
    • 보는 바위를 이긴다 📄 > ✊
    • 같으면 무승부

    📖 정확한 정의

    승패 판정은 사용자의 선택과 컴퓨터의 선택을 비교하여, 미리 정해진 규칙에 따라 승/패/무를 결정하는 로직입니다. 조건문(if-else)으로 구현합니다.

    💻 예시로 확인

    // 판정 규칙을 코드로 표현
if (user === computer) {
    return "무승부";
} else if (
    (user === "가위" && computer === "보") ||
    (user === "바위" && computer === "가위") ||
    (user === "보" && computer === "바위")
) {
    return "승리";
} else {
    return "패배";
}

    핵심은 사용자가 이기는 3가지 경우를 명시하는 것입니다. 나머지는 자동으로 패배가 되니까요.

    개념 4: 전적 기록과 누적¶

    🎨 비유로 시작

    스포츠 경기를 볼 때, 스코어보드에 점수가 계속 쌓이잖아요? 우리 게임에서도 승리 횟수, 패배 횟수, 무승부 횟수를 스코어보드처럼 기록해 나갑니다.

    📖 정확한 정의

    게임이 진행될 때마다 결과에 따라 해당 카운터 변수를 1씩 증가시키는 것을 전적 누적이라고 합니다. 게임이 끝나면 이 값들을 모아 최종 전적을 보여줍니다.

    💻 예시로 확인

    let wins = 0;    // 승리 횟수
let losses = 0;  // 패배 횟수
let draws = 0;   // 무승부 횟수

// 게임 결과에 따라 누적
if (result === "승리") {
    wins++;       // wins = wins + 1 과 같은 뜻
} else if (result === "패배") {
    losses++;
} else {
    draws++;
}
    flowchart LR A["게임 1회 진행"] --> B{"결과는?"} B -->|승리| C["wins + 1"] B -->|패배| D["losses + 1"] B -->|무승부| E["draws + 1"] C --> F["다음 게임 또는 종료"] D --> F E --> F

    💡 잠깐! 여기까지 4가지 핵심 개념을 살펴봤어요. 이제부터 이 개념들을 하나씩 조립하면서 실제 게임을 만들어 봅니다. 설레지 않나요? 😊

    🔨 따라하기¶

    이제 본격적으로 코드를 작성합니다! 총 6단계로 나누어 진행하며, 매 단계마다 실행해서 확인합니다.

    🛠️ 준비물: 웹 브라우저(Chrome 추천)의 개발자 도구 콘솔, 또는 온라인 코딩 환경(replit.com 등)

    이번 실습에서는 prompt()와 alert() 대신 콘솔 기반으로 먼저 만든 후, 마지막에 prompt()/alert() 버전도 안내합니다.

    Step 1: 컴퓨터의 선택을 만드는 함수¶

    목표: 컴퓨터가 가위/바위/보 중 하나를 랜덤으로 고르는 함수를 만듭니다.

    // === Step 1: 컴퓨터 선택 함수 ===

function getComputerChoice() {
    let choices = ["가위", "바위", "보"];
    let randomIndex = Math.floor(Math.random() * 3);
    return choices[randomIndex];
}

// 테스트: 5번 실행해서 잘 되는지 확인!
console.log("컴퓨터 선택:", getComputerChoice());
console.log("컴퓨터 선택:", getComputerChoice());
console.log("컴퓨터 선택:", getComputerChoice());
console.log("컴퓨터 선택:", getComputerChoice());
console.log("컴퓨터 선택:", getComputerChoice());

    ✅ 체크포인트: 실행할 때마다 "가위", "바위", "보" 중 하나가 랜덤으로 출력되어야 해요. 5번 중에 적어도 2가지 이상 다른 결과가 나오면 정상입니다!

    이 코드가 하는 일을 한 줄씩 살펴볼게요:

    • let choices = ["가위", "바위", "보"]; → 선택지 3개를 배열에 담습니다
    • Math.random() * 3 → 0 이상 3 미만의 소수를 만듭니다
    • Math.floor(...) → 소수점을 버려서 0, 1, 2 중 하나의 정수를 얻습니다
    • choices[randomIndex] → 배열에서 해당 번째 값을 꺼냅니다
    • return → 함수를 호출한 곳으로 결과를 돌려줍니다
    flowchart TD A["Math.random()"] -->|"예: 0.742"| B["× 3"] B -->|"예: 2.226"| C["Math.floor()"] C -->|"예: 2"| D["choices 배열에서 2번 인덱스"] D -->|"'보'"| E["return '보'"]

    Step 2: 승패 판정 함수¶

    목표: 사용자와 컴퓨터의 선택을 받아서 "승리", "패배", "무승부"를 돌려주는 함수를 만듭니다.

    // === Step 2: 승패 판정 함수 ===

function judge(userChoice, computerChoice) {
    if (userChoice === computerChoice) {
        return "무승부";
    } else if (
        (userChoice === "가위" && computerChoice === "보") ||
        (userChoice === "바위" && computerChoice === "가위") ||
        (userChoice === "보" && computerChoice === "바위")
    ) {
        return "승리";
    } else {
        return "패배";
    }
}

// 테스트: 다양한 경우를 직접 넣어서 확인!
console.log(judge("가위", "보"));    // "승리"가 나와야 함
console.log(judge("가위", "바위"));  // "패배"가 나와야 함
console.log(judge("가위", "가위"));  // "무승부"가 나와야 함
console.log(judge("바위", "가위"));  // "승리"가 나와야 함
console.log(judge("보", "바위"));    // "승리"가 나와야 함
console.log(judge("보", "가위"));    // "패배"가 나와야 함

    ✅ 체크포인트: 위 6가지 테스트 결과가 주석과 일치하면 성공! 특히 모든 승리 조건 3개가 다 "승리"로 나오는지 꼭 확인하세요.

    왜 이렇게 테스트를 많이 하나요?

    승패 판정은 게임의 핵심입니다. 여기서 실수가 있으면 게임 전체가 엉뚱하게 동작해요. "가위가 바위를 이긴다"고 잘못 코딩하면... 그건 가위바위보가 아니라 이상한 게임이 되겠죠? 😅 그래서 가능한 모든 경우를 테스트하는 거예요. 이것을 개발자들은 "테스트 케이스"라고 부릅니다.

    💡 잠깐! Step 1과 Step 2까지 완성했다면 이미 절반을 넘었어요! 핵심 로직 두 개를 만든 거거든요. 나머지는 이 두 함수를 연결하고 꾸미는 작업입니다. 👏

    Step 3: 한 판 플레이 함수¶

    목표: Step 1과 Step 2를 합쳐서, 한 판의 게임을 진행하는 함수를 만듭니다.

    // === Step 3: 한 판 플레이 함수 ===
// (Step 1, 2의 함수가 위에 있다고 가정)

function playRound(userChoice) {
    // 1. 컴퓨터 선택
    let computerChoice = getComputerChoice();

    // 2. 승패 판정
    let result = judge(userChoice, computerChoice);

    // 3. 결과 출력
    console.log("--------------------");
    console.log("🧑 나: " + userChoice);
    console.log("🤖 컴퓨터: " + computerChoice);

    if (result === "승리") {
        console.log("🎉 결과: 승리!");
    } else if (result === "패배") {
        console.log("😢 결과: 패배...");
    } else {
        console.log("🤝 결과: 무승부!");
    }
    console.log("--------------------");

    // 4. 결과 값을 돌려줌 (나중에 전적 기록에 사용)
    return result;
}

// 테스트: 직접 선택을 넣어서 한 판 해보기!
playRound("가위");
playRound("바위");
playRound("보");

    ✅ 체크포인트: 실행하면 아래와 비슷한 출력이 나와야 해요 (컴퓨터 선택은 랜덤이라 결과는 다를 수 있어요): 

    --------------------

    여기서 포인트!

    playRound 함수는 getComputerChoice와 judge 함수를 내부에서 호출합니다. 함수가 함수를 부르는 거예요. 마치 요리사가 재료 준비 담당자에게 "양파 좀 줘!"라고 말하고, 간 보기 담당자에게 "이거 간 맞아?"라고 물어보는 것과 같아요.

    Step 4: 전적 기록 기능 추가¶

    목표: 여러 판을 기록할 수 있도록 전적 변수를 만들고, 결과에 따라 누적합니다.

    // === Step 4: 전적 기록 추가 ===
// (Step 1, 2, 3의 함수가 위에 있다고 가정)

// 전적 변수 선언
let wins = 0;
let losses = 0;
let draws = 0;

// 전적을 업데이트하는 함수
function updateRecord(result) {
    if (result === "승리") {
        wins++;
    } else if (result === "패배") {
        losses++;
    } else {
        draws++;
    }
}

// 전적을 보여주는 함수
function showRecord() {
    console.log("====================");
    console.log("📊 현재 전적");
    console.log("  승리: " + wins + "회");
    console.log("  패배: " + losses + "회");
    console.log("  무승부: " + draws + "회");
    console.log("  총 " + (wins + losses + draws) + "판 진행");
    console.log("====================");
}

// 테스트: 수동으로 3판 진행해 보기
let result1 = playRound("가위");
updateRecord(result1);

let result2 = playRound("바위");
updateRecord(result2);

let result3 = playRound("보");
updateRecord(result3);

// 전적 확인!
showRecord();

    ✅ 체크포인트: showRecord() 실행 결과에서 승리 + 패배 + 무승부 = 3이면 정상! 전적이 제대로 누적되고 있는 거예요.

    ++ 연산자 복습:

    wins++는 wins = wins + 1과 똑같은 뜻입니다. "현재 값에서 1을 더해서 다시 저장해라"는 의미예요. 게임에서 점수를 1점씩 올리는 것과 같습니다.

    💡 여기까지 잘 따라오셨나요? 벌써 게임의 핵심 기능이 다 완성되었어요! 남은 건 사용자 입력을 받는 것과 반복 플레이 기능뿐입니다. 거의 다 왔어요! 💪

    Step 5: 사용자 입력과 반복 플레이¶

    목표: prompt()로 사용자 입력을 받고, "다시 하시겠습니까?" 기능으로 반복 플레이를 구현합니다.

    // === Step 5: 사용자 입력 + 반복 플레이 ===

function getUserChoice() {
    let input = prompt("가위, 바위, 보 중 하나를 입력하세요:");

    // 입력값 확인 (잘못된 입력 방지)
    if (input !== "가위" && input !== "바위" && input !== "보") {
        alert("⚠️ '가위', '바위', '보' 중 하나를 정확히 입력해주세요!");
        return null;  // 잘못된 입력이면 null 반환
    }

    return input;
}

// 게임 메인 함수
function startGame() {
    // 전적 초기화
    wins = 0;
    losses = 0;
    draws = 0;

    let playing = true;

    while (playing) {
        // 1. 사용자 입력 받기
        let userChoice = getUserChoice();

        // 잘못된 입력이면 이번 판 건너뛰기
        if (userChoice === null) {
            continue;  // while문의 처음으로 돌아감
        }

        // 2. 한 판 진행
        let result = playRound(userChoice);

        // 3. 전적 업데이트
        updateRecord(result);

        // 4. 계속할지 물어보기
        let answer = prompt("다시 하시겠습니까? (예/아니오)");
        if (answer !== "예") {
            playing = false;  // 반복 종료
        }
    }

    // 5. 최종 전적 출력
    console.log("\n🎮 게임 종료!");
    showRecord();
    alert("게임이 종료되었습니다! 콘솔에서 전적을 확인하세요.");
}

    flowchart TD A["startGame() 시작"] --> B["전적 초기화"] B --> C["while 반복문 시작"] C --> D["getUserChoice()로 입력 받기"] D --> E{"입력이 올바른가?"} E -->|"아니오 (null)"| C E -->|"예"| F["playRound()로 한 판 진행"] F --> G["updateRecord()로 전적 업데이트"] G --> H{"다시 하시겠습니까?"} H -->|"예"| C H -->|"아니오"| I["showRecord()로 최종 전적 출력"] I --> J["게임 종료 🎮"]

    continue란?

    continue는 반복문 안에서 "이번 회차는 여기서 그만하고, 다시 처음부터!"라는 뜻입니다. 잘못된 입력이 들어왔을 때, 그 입력으로 게임을 진행하지 않고 다시 입력을 받기 위해 사용해요.

    null이란?

    null은 "값이 없음"을 의미하는 특별한 값입니다. 여기서는 "올바른 입력을 받지 못했다"는 신호로 사용하고 있어요.
