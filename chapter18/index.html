    보너스 도전! OX 퀴즈 게임 만들기¶

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • 배열(Array)에 퀴즈 데이터를 저장하고 순서대로 출제하는 로직을 구현할 수 있다
    • 정답 확인, 점수 계산, 결과 발표 기능을 각각 함수로 분리할 수 있다
    • 스스로 설계 → 구현 → 테스트 과정을 수행할 수 있다
    • 사용자 입력을 검증하고 예외 상황을 처리할 수 있다

    예상 학습 시간: 약 2시간 30분

    📚 미리 알아야 할 것¶

    이전 장에서 가위바위보 게임을 만들면서 배운 내용을 간단히 복습해 볼게요.

    • 변수와 자료형: 숫자, 문자열을 저장하는 방법
    • 조건문 (if/else): 조건에 따라 다른 코드를 실행하는 방법
    • 반복문 (for): 같은 작업을 여러 번 반복하는 방법
    • 함수: 코드를 묶어서 이름을 붙이고 재사용하는 방법
    • 배열 기초: 여러 값을 하나의 변수에 담는 방법
    • prompt/alert: 사용자에게 입력을 받고 결과를 보여주는 방법

    💡 잠깐! 위 내용이 기억나지 않더라도 걱정 마세요. 이번 장에서 필요한 부분은 그때그때 다시 설명해 드릴게요!

    💡 왜 이걸 배우나요?¶

    가위바위보 게임을 성공적으로 완성한 여러분, 정말 대단해요! 🎉

    그런데 한 가지 생각해 볼까요? 가위바위보는 선택지가 3개이고, 문제가 1개인 단순한 구조였어요. 실제로 우리가 쓰는 앱이나 게임은 여러 개의 데이터를 다루고, 반복적으로 처리하는 경우가 훨씬 많죠.

    OX 퀴즈 게임을 만들면서 이런 능력을 키울 수 있어요:

    • 여러 문제를 배열로 관리하기 → 데이터를 구조화하는 기초
    • 기능별로 함수를 나누기 → 프로그램을 설계하는 습관
    • 혼자서 처음부터 끝까지 만들기 → 진짜 개발자의 경험

    🎮 완성 모습 미리보기: "다음 중 맞으면 O, 틀리면 X를 입력하세요!" → 5문제 출제 → 정답 확인 → 최종 점수 발표!

    이번에는 가위바위보 때보다 힌트를 줄이고, 여러분이 직접 생각하는 비중을 높였어요. 막히면 단계별 힌트가 있으니 안심하세요. 스스로 고민하고 해결하는 그 과정이 진짜 실력이 됩니다! 💪

    📚 핵심 개념¶

    이번 장에서 집중할 핵심 개념은 딱 4가지입니다.

    개념 1: 배열로 퀴즈 데이터 관리하기¶

    🍱 비유로 시작¶

    도시락 가게를 떠올려 보세요. 메뉴판에 1번부터 5번까지 도시락이 적혀 있고, 손님이 번호를 말하면 해당 도시락을 꺼내줍니다. 배열도 마찬가지예요. 번호(인덱스)로 원하는 데이터를 꺼내 쓸 수 있는 정리된 서랍장이에요.

    📖 정확한 정의¶

    배열(Array)은 여러 개의 값을 순서대로 하나의 변수에 저장하는 자료구조입니다. OX 퀴즈에서는 문제 목록과 정답 목록을 각각 배열에 담아서 관리합니다.

    💻 예시로 확인¶

    // 퀴즈 문제를 배열에 저장
let questions = [
  "HTML은 프로그래밍 언어이다.",
  "JavaScript는 웹 브라우저에서 동작한다.",
  "CSS는 웹페이지의 구조를 담당한다."
];

// 정답을 배열에 저장 (문제 순서와 동일)
let answers = ["X", "O", "X"];

// 첫 번째 문제 꺼내기 (인덱스 0부터 시작!)
console.log(questions[0]); // "HTML은 프로그래밍 언어이다."
console.log(answers[0]);   // "X"

    여기서 핵심은 questions[0]과 answers[0]이 짝이라는 점이에요. 0번 문제의 정답은 answers[0]에 있습니다.

    graph LR subgraph questions 배열 Q0["[0] HTML은 프로그래밍 언어이다."] Q1["[1] JavaScript는 웹 브라우저에서 동작한다."] Q2["[2] CSS는 웹페이지의 구조를 담당한다."] end subgraph answers 배열 A0["[0] X"] A1["[1] O"] A2["[2] X"] end Q0 -. "짝" .-> A0 Q1 -. "짝" .-> A1 Q2 -. "짝" .-> A2

    💡 포인트: 두 배열의 같은 인덱스끼리 문제-정답이 연결됩니다. 이 구조를 이해하면 퀴즈 게임의 80%는 완성이에요!

    개념 2: 반복문으로 순서대로 출제하기¶

    📻 비유로 시작¶

    라디오 DJ가 플레이리스트의 노래를 1번부터 마지막까지 차례대로 틀어주는 것처럼, 반복문(for)을 사용하면 배열의 문제를 첫 번째부터 마지막까지 순서대로 출제할 수 있어요.

    📖 정확한 정의¶

    for 반복문은 시작값, 조건, 증가값을 설정하여 배열의 모든 요소를 순서대로 처리합니다. 배열의 길이(length)를 활용하면 문제 수가 바뀌어도 코드를 수정할 필요가 없어요.

    💻 예시로 확인¶

    let questions = [
  "HTML은 프로그래밍 언어이다.",
  "JavaScript는 웹 브라우저에서 동작한다.",
  "CSS는 웹페이지의 구조를 담당한다."
];

// questions.length는 3 → i가 0, 1, 2일 때 반복
for (let i = 0; i < questions.length; i++) {
  console.log("문제 " + (i + 1) + ": " + questions[i]);
}

    실행 결과:

    문제 1: HTML은 프로그래밍 언어이다.
문제 2: JavaScript는 웹 브라우저에서 동작한다.
문제 3: CSS는 웹페이지의 구조를 담당한다.

    🔑 핵심: i는 0부터 시작하지만, 사용자에게 보여줄 때는 i + 1로 표시해서 1번 문제부터 시작하게 해요. 사람은 1부터 세니까요!

    개념 3: 기능별 함수 분리¶

    🏭 비유로 시작¶

    공장에서 자동차를 만들 때, 한 사람이 모든 걸 하지 않아요. 도색팀, 엔진팀, 검수팀이 각자 맡은 일을 합니다. 프로그래밍도 마찬가지예요. 전체 코드를 한 덩어리로 쓰는 대신, 역할별로 함수를 나누면 읽기도 쉽고 고치기도 쉬워요.

    📖 정확한 정의¶

    함수 분리란 프로그램의 기능을 독립적인 함수로 나누는 것입니다. 각 함수는 하나의 역할만 담당하며, 필요할 때 호출해서 사용합니다. 이번 퀴즈 게임에서는 세 가지 함수로 나눌 거예요.

    💻 예시로 확인¶

    // 1. 문제를 출제하고 답을 받는 함수
function askQuestion(questionText) {
  let userAnswer = prompt("OX 퀴즈!\n\n" + questionText + "\n\nO 또는 X를 입력하세요:");
  return userAnswer;
}

// 2. 정답을 확인하는 함수
function checkAnswer(userAnswer, correctAnswer) {
  if (userAnswer === correctAnswer) {
    return true;   // 맞음
  } else {
    return false;  // 틀림
  }
}

// 3. 최종 결과를 발표하는 함수
function showResult(score, total) {
  alert("퀴즈 끝! " + total + "문제 중 " + score + "문제 정답!");
}
    graph TD A["🎮 게임 시작"] --> B["askQuestion<br/>문제 출제 + 답 입력받기"] B --> C["checkAnswer<br/>정답 확인"] C --> D{모든 문제<br/>출제 완료?} D -- "아직 남음" --> B D -- "다 끝남" --> E["showResult<br/>최종 결과 발표"] E --> F["🏁 게임 종료"]

    💡 왜 이렇게 나누나요? 나중에 "결과를 더 예쁘게 보여주고 싶다"면 showResult 함수만 수정하면 돼요. 다른 부분은 건드릴 필요가 없죠. 이것이 함수 분리의 힘입니다!

    개념 4: 입력값 검증 (유효성 검사)¶

    🚪 비유로 시작¶

    놀이공원 입구에서 키를 재는 것과 같아요. 키가 120cm 미만이면 "이 놀이기구는 탈 수 없어요"라고 안내해 주죠. 프로그램에서도 사용자가 이상한 값을 입력하면 "다시 입력해 주세요"라고 알려줘야 해요.

    📖 정확한 정의¶

    입력값 검증이란 사용자가 입력한 값이 프로그램이 기대하는 형식에 맞는지 확인하는 과정입니다. OX 퀴즈에서는 "O" 또는 "X"만 유효한 입력이에요. 그 외의 값(예: "A", "ㅇ", 빈 문자열)이 들어오면 다시 입력받아야 합니다.

    💻 예시로 확인¶

    function getValidInput(questionText) {
  let input = prompt("OX 퀴즈!\n\n" + questionText + "\n\nO 또는 X를 입력하세요:");

  // 입력값을 대문자로 변환 (소문자 o, x도 허용하기 위해)
  input = input.toUpperCase();

  // O 또는 X가 아니면 다시 입력받기
  while (input !== "O" && input !== "X") {
    input = prompt("⚠️ O 또는 X만 입력해주세요!\n\n" + questionText);
    input = input.toUpperCase();
  }

  return input;
}

    🛡️ toUpperCase()란? 소문자를 대문자로 바꿔주는 기능이에요. "o".toUpperCase()는 "O"가 됩니다. 이렇게 하면 사용자가 소문자로 입력해도 잘 처리할 수 있어요!

    힌트 1 - askQuestion 함수:

    function askQuestion(number, questionText) {
  let input = prompt(
    "📝 문제 " + number + "\n\n" +
    questionText + "\n\n" +
    "O 또는 X를 입력하세요:"
  );

  // null 체크 (사용자가 취소를 누른 경우)
  if (input === null) {
    input = "";
  }

  // 대문자로 변환
  input = input.toUpperCase();

  // O 또는 X가 아니면 다시 입력받기
  while (input !== "O" && input !== "X") {
    input = prompt(
      "⚠️ O 또는 X만 입력해주세요!\n\n" +
      "📝 문제 " + number + "\n" +
      questionText
    );

    if (input === null) {
      input = "";
    }
    input = input.toUpperCase();
  }

  return input;
}

    💡 input === null 체크는 뭐예요? prompt 대화상자에서 "취소" 버튼을 누르면 null이 반환돼요. null에 toUpperCase()를 쓰면 에러가 나기 때문에 미리 처리해 주는 거예요!

    function showResult(score, total) {
  let message = "🏆 퀴즈 결과 🏆\n\n";
  message += total + "문제 중 " + score + "문제 정답!\n\n";

  // 점수에 따른 등급
  if (score === total) {
    message += "🥇 완벽해요! 천재인가요?!";
  } else if (score >= total * 0.8) {
    message += "🥈 훌륭해요! 거의 다 맞혔어요!";
  } else if (score >= total * 0.6) {
    message += "🥉 잘했어요! 조금만 더 공부하면 완벽!";
  } else if (score >= total * 0.4) {
    message += "💪 괜찮아요! 다시 도전해 보세요!";
  } else {
    message += "📚 조금 더 공부가 필요해요! 힘내세요!";
  }

  alert(message);
}

    게임의 흐름을 다시 떠올려 보세요:

    1. 환영 메시지를 보여준다
    2. 점수를 0으로 초기화한다
    3. for문으로 문제를 하나씩 출제한다
    4. 각 문제마다 정답을 확인하고, 맞으면 점수를 올린다
    5. 맞았는지 틀렸는지 알려준다
    6. 모든 문제가 끝나면 결과를 발표한다

    이 흐름을 코드로 작성해 보세요!

    // ===== 게임 시작 =====
function startQuiz() {
  // 1. 환영 메시지
  alert("🧠 OX 퀴즈 게임에 오신 것을 환영합니다!\n\n총 " + questions.length + "문제가 출제됩니다.\n\n맞으면 O, 틀리면 X를 입력하세요!");

  // 2. 점수 초기화
  let score = 0;

  // 3. 문제 출제 (반복문)
  for (let i = 0; i < questions.length; i++) {
    // 3-1. 문제 출제하고 답 받기
    let userAnswer = askQuestion(i + 1, questions[i]);

    // 3-2. 정답 확인
    let isCorrect = checkAnswer(userAnswer, answers[i]);

    // 3-3. 결과 알려주기
    if (isCorrect) {
      alert("⭕ 정답입니다! 👏");
      score = score + 1;
    } else {
      alert("❌ 틀렸습니다! 정답은 " + answers[i] + "입니다.");
    }
  }

  // 4. 최종 결과 발표
  showResult(score, questions.length);
}

// 게임 실행!
startQuiz();
