    Chapter 01: 왜 C언어를 배워야 할까? — 이 책의 사용법¶

    🎯 이 장의 학습 목표¶

    이 장을 마치면 다음을 할 수 있습니다:

    • 프로그래밍이 무엇인지 자신의 말로 설명할 수 있다
    • 이 책의 전체 학습 로드맵과 2주 완주 계획을 이해할 수 있다
    • 최종 목표인 미니 게임의 완성 모습을 미리 살펴보고 기대감을 가질 수 있다

    예상 학습 시간: 약 50~60분

    📚 미리 알아야 할 것¶

    아무것도 몰라도 괜찮아요! 🎉 이 장은 진짜 '제로'에서 시작합니다. 컴퓨터를 켜고, 키보드를 칠 수 있다면 충분해요.

    📚 핵심 개념¶

    개념 1: 프로그래밍이란 무엇일까?¶

    비유로 시작해 볼게요. 🍳

    여러분이 친구에게 볶음밥을 만들어 달라고 부탁한다고 상상해 보세요. 그냥 "볶음밥 만들어줘"라고 하면, 친구는 어떤 재료를 쓸지, 불은 얼마나 세게 할지, 언제 밥을 넣을지 모릅니다. 그래서 우리는 레시피를 줍니다.

    • 1단계: 팬에 기름을 두른다
    • 2단계: 파를 넣고 30초 볶는다
    • 3단계: 밥을 넣고 2분 볶는다
    • 4단계: 간장을 한 스푼 넣고 섞는다

    프로그래밍은 바로 이 레시피를 작성하는 것과 같아요. 다만 요리사가 아니라 컴퓨터에게 주는 레시피라는 점이 다릅니다.

    📖 정확한 정의: 프로그래밍이란, 컴퓨터가 특정 작업을 수행하도록 명령어를 순서대로 작성하는 것입니다. 이렇게 작성된 명령어 모음을 프로그램이라고 부릅니다.

    컴퓨터는 아주 빠르고 정확하지만, 스스로 생각하지는 못해요. 그래서 우리가 하나하나 정확하게 알려줘야 합니다. "대충 볶아"라고 하면 안 되고, "2분 동안 중불로 볶아"라고 해야 하는 거죠.

    예시로 확인해 볼까요? 여러분이 매일 사용하는 것들이 모두 프로그래밍의 결과물입니다:

    • 📱 카카오톡 — 누군가 "메시지를 보내면 상대방 화면에 표시해라"라고 프로그래밍한 것
    • 🎮 게임 — "캐릭터가 점프 버튼을 누르면 위로 올라가라"라고 프로그래밍한 것
    • 🧮 계산기 앱 — "숫자 두 개와 연산자를 입력받으면 결과를 보여라"라고 프로그래밍한 것

    💡 잠깐! "나는 문과인데 할 수 있을까?" 걱정되시나요? 프로그래밍은 수학 천재만 하는 게 아니에요. 레시피를 쓸 수 있으면 프로그래밍도 할 수 있습니다. 천천히 따라오시면 돼요!

    개념 2: 왜 하필 C언어일까?¶

    프로그래밍 언어는 수백 가지가 있습니다. Python, Java, JavaScript… 이 많은 언어 중에서 왜 C언어부터 배울까요?

    비유로 시작해 볼게요. 🌳

    C언어는 프로그래밍 언어의 뿌리와 같아요. 거대한 나무를 떠올려 보세요. C언어가 뿌리와 줄기라면, Java·Python·C++ 같은 현대 언어들은 가지와 잎사귀입니다.

    graph TD C["🌱 C언어 (1972년 탄생)"] C --> CPP["C++ (게임, 시스템)"] C --> JAVA["Java (앱, 서버)"] C --> CSHARP["C# (게임, 윈도우)"] C --> PYTHON["Python에도 영향"] style C fill:#4CAF50,stroke:#333,color:#fff style CPP fill:#2196F3,stroke:#333,color:#fff style JAVA fill:#FF9800,stroke:#333,color:#fff style CSHARP fill:#9C27B0,stroke:#333,color:#fff style PYTHON fill:#FF5722,stroke:#333,color:#fff

    📖 정확한 정의: C언어는 1972년 데니스 리치(Dennis Ritchie)가 만든 프로그래밍 언어로, 컴퓨터의 작동 원리를 가장 가까이에서 배울 수 있는 언어입니다.

    C언어를 배우면 좋은 이유 3가지:

    • 🏗️ 기초 체력이 생겨요 — C를 알면 다른 언어를 배울 때 훨씬 수월합니다
    • ⚡ 컴퓨터를 깊이 이해해요 — 메모리, 처리 과정 등 컴퓨터의 속사정을 알게 됩니다
    • 🌍 어디서나 쓰여요 — 운영체제(Windows, Linux), 게임 엔진, 로봇, IoT 기기까지!

    💡 잠깐! "C언어가 오래됐으니까 쓸모없는 거 아냐?"라고 생각할 수 있어요. 하지만 2024년 현재에도 C언어는 프로그래밍 언어 인기 순위 TOP 3 안에 늘 들어가는 현역 언어랍니다! 💪

    개념 3: 이 책의 로드맵 — 어디로 가고 있는지 알면 힘이 나요¶

    여행을 떠나기 전에 지도를 한번 펼쳐보는 것처럼, 우리 학습의 전체 그림을 먼저 살펴볼게요.

    graph LR P1["📗 1권<br/>첫 발 내딛기<br/>(1~5차시)"] P2["📘 2권<br/>문법 탐험<br/>(6~10차시)"] P3["📙 3권<br/>실력 키우기<br/>(11~15차시)"] P4["📕 4권<br/>프로젝트 완성<br/>(16~20차시)"] P1 --> P2 --> P3 --> P4 style P1 fill:#E8F5E9,stroke:#4CAF50,color:#333 style P2 fill:#E3F2FD,stroke:#2196F3,color:#333 style P3 fill:#FFF3E0,stroke:#FF9800,color:#333 style P4 fill:#FFEBEE,stroke:#F44336,color:#333

    각 권에서 배우는 내용:

    • 📗 1권 (1~5차시): 프로그래밍 첫걸음 — 화면에 글자 출력하기, 변수, 입출력
    • 📘 2권 (6~10차시): 문법 탐험 — 조건문, 반복문, 함수
    • 📙 3권 (11~15차시): 실력 키우기 — 배열, 문자열, 포인터 기초
    • 📕 4권 (16~20차시): 프로젝트 완성 — 미니 게임을 직접 만들기!

    2주 완주 계획 🗓️

    하루에 딱 1시간, 평일에만 공부하면 2주면 완주할 수 있어요!

    • 1주차 월~금: 1차시 → 5차시 (1권 완료! 🎉)
    • 주말: 쉬면서 복습
    • 2주차 월~금: 6차시 → 10차시 (2권 완료! 🎉)
    • 이후 같은 패턴으로: 3권, 4권 완주

    물론 자신의 페이스에 맞춰 조절해도 전혀 괜찮아요. 꾸준히 하는 것이 가장 중요합니다! 🐢

    🔨 따라하기: 최종 프로젝트 미리 구경하기¶

    아직 코드를 작성하는 건 아니에요. 우리가 20차시 뒤에 직접 만들게 될 프로그램이 어떻게 생겼는지 먼저 구경해 봅시다! 🎮

    Step 1: 숫자 맞추기 게임 완성 모습¶

    이 책을 끝까지 따라오면, 이런 게임을 스스로 만들 수 있게 됩니다:

    실행 결과: 

    === 숫자 맞추기 게임 ===
1부터 100 사이의 숫자를 맞춰보세요!

숫자를 입력하세요: 50
너무 커요! 더 작은 수를 시도하세요.

숫자를 입력하세요: 25
너무 작아요! 더 큰 수를 시도하세요.

숫자를 입력하세요: 37
🎉 정답입니다! 3번 만에 맞추셨어요!

    컴퓨터가 몰래 숫자를 하나 정하고, 여러분이 맞추는 게임이에요. 힌트도 주고요!

    Step 2: 가위바위보 게임 완성 모습¶

    두 번째로 만들 게임도 살짝 엿볼게요:

    실행 결과: 

    === 가위바위보 게임 ===
1: 가위  2: 바위  3: 보

선택하세요: 2
당신: 바위 vs 컴퓨터: 가위
🎉 당신이 이겼습니다!

계속하시겠습니까? (1:예 / 0:아니오): 0
전적: 1승 0패 0무  승률: 100%

    ✅ 체크포인트: "와, 이런 걸 내가 만들 수 있다고?" 하는 느낌이 드시나요? 그 느낌이 맞습니다! 이 책을 차근차근 따라오면 반드시 만들 수 있어요.

    Step 3: 우리가 가장 먼저 작성할 코드 맛보기¶

    다음 차시(2차시)에서 우리가 가장 먼저 작성할 코드를 살짝 보여드릴게요.

    코드: 

    #include <stdio.h>

int main(void) {
    printf("Hello, World!\n");  // 화면에 글자를 출력하는 명령
    return 0;
}

    실행 결과: 

    Hello, World!

    지금은 이 코드가 외계어처럼 보여도 전혀 걱정 마세요. 다음 차시에서 한 줄 한 줄 친절하게 설명할 거예요. 지금은 "아, 이 정도 분량의 코드로 화면에 글자가 나오는구나" 정도만 느끼시면 됩니다. 😊

    ⚠️ 주의할 점¶

    1. "이해 안 되면 나만 이상한 건가?" — 아닙니다! 🙅

    프로그래밍은 처음에 누구나 낯설어요. 한 번 읽어서 모르면 두 번 읽으면 됩니다. 세 번 읽어도 괜찮아요. 프로 개발자들도 처음엔 다 그랬답니다.

    2. 타이핑은 직접 하세요! ⌨️

    코드를 눈으로만 읽지 말고, 반드시 직접 타이핑해 보세요. 복사-붙여넣기보다 직접 치는 것이 학습 효과가 3배 이상 높다는 연구 결과도 있어요.

    ✅ 점검하기¶

    스스로 점검해 봅시다! 아래 질문에 답할 수 있으면 이 장은 완벽히 끝낸 거예요. 🎯

    1. 프로그래밍을 한 마디로 설명하면?

    정답 확인 프로그래밍은 **컴퓨터에게 일을 시키기 위해 명령어를 순서대로 작성하는 것**입니다. 요리 레시피처럼, 컴퓨터가 따라야 할 단계별 지시서를 만드는 거예요.

    2. C언어를 배우면 좋은 이유를 2가지 이상 말해보세요.

    정답 확인 - 다른 프로그래밍 언어(C++, Java 등)의 **뿌리**이기 때문에, C를 알면 다른 언어를 배우기 쉬워요 - **컴퓨터의 작동 원리**(메모리, 처리 과정)를 깊이 이해할 수 있어요 - 운영체제, 게임, 로봇 등 **다양한 분야에서 현재도 널리 사용**되고 있어요

    3. 이 책은 총 몇 차시이며, 최종적으로 무엇을 만드나요?

    정답 확인 총 **20차시**이며, 최종적으로 **숫자 맞추기 게임**과 **가위바위보 게임**을 직접 만듭니다!

    📌 이 장의 핵심 정리¶

    • ✅ 프로그래밍 = 컴퓨터에게 주는 명령서(레시피)를 작성하는 것
    • ✅ C언어 = 프로그래밍 언어의 뿌리, 배우면 다른 언어도 쉬워진다
    • ✅ 이 책 = 20차시, 하루 1시간, 2주면 미니 게임을 만들 수 있다
